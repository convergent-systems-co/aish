## Acceptance Criteria

### #87 — Remaining built-ins (`alias`, `source`, `set`, `unset`)

- [ ] `alias NAME=COMMAND` registers an alias on the live `Shell`; bare
      `alias` lists current aliases (sorted) in `NAME='COMMAND'` form;
      `alias NAME` prints that one entry or exits 1.
- [ ] Aliases declared in the RC `[aliases]` table (already parsed by
      v0.3-1 #190) apply at startup — the dispatcher rewrites the first
      token of each command line when it matches an alias key.
- [ ] Alias expansion happens **after** brace/glob/cmdsub expansion and
      **before** known-binary / cache dispatch; recursion is bounded
      (16-step cap) so `alias x=y; alias y=x` cannot loop.
- [ ] `source <file>` reads a TOML aishrc (same schema as login RC) or
      POSIX env-only file (lines of `K=V`) into the live shell. On
      missing file: stderr + exit 1; on parse error: stderr + exit 1.
- [ ] `set NAME=VALUE` binds in the local env (no export flag). Bare
      `set` lists all session env entries (sorted) — matches POSIX
      "list variables" semantics. We do NOT implement option flags
      (`set -e`, `set -o`) in this PR; they're out of scope.
- [ ] `unset NAME` removes the binding from env. Empty / missing arg
      prints usage and exits 2.
- [ ] Tab-completion + `ResolveTier` recognize all four new built-ins.

### #88 — Expansion (brace, glob, command substitution)

- [ ] **Brace expansion**: `{a,b,c}` → three tokens `a b c`. Numeric
      ranges `{1..5}` → `1 2 3 4 5`. Combined with prefix/suffix:
      `pre{a,b}post` → `preapost prebpost`. Nested braces not
      required this PR (bash supports them; we emit a TODO comment).
- [ ] **Glob expansion**: `*.go`, `cmd/*/main.go`, `???.txt` — every
      bare token containing `*`, `?`, or `[` is matched against the
      filesystem via `path/filepath.Glob`, rooted at the shell's
      cwd. No match → token preserved literally (POSIX
      `nullglob`-off default; matches bash without `failglob`).
- [ ] **Command substitution**: `$(cmd)` runs `cmd` via the shell's
      own pipeline; the child's stdout (trailing newlines trimmed)
      becomes the substituted text. Recursive (a `$(...)` inside
      another `$(...)` is supported). Backticks intentionally NOT
      supported in this PR.
- [ ] Expansion order matches POSIX where it matters: `$VAR` first
      (existing `env.Expand`), then `$(cmd)`, then brace, then glob,
      then alias rewrite. Inside single quotes nothing expands;
      inside double quotes only `$VAR` and `$(cmd)` expand.
- [ ] Live smoke (`make ci` + manual): `alias ll='ls -la'`, `ll`,
      `source /tmp/some.aishrc`, `echo {a,b,c}` → `a b c`,
      `echo *.go` expands, `echo $(date)` substitutes.

## Rationale + Alternatives

### Brace expansion: in-parser vs post-parser pass

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **Post-parser pass over Pipeline tokens** | Parser stays single-purpose. Expansion is testable in isolation. Easy to chain with glob/cmdsub. | Token boundaries are decided by parser, then re-walked. Slight redundancy. | **Chosen** |
| In-parser, expand during tokenize | One pass. | Tangles three independent expansions into the tokenizer. Glob needs cwd; cmdsub needs a Shell handle — both poison the parser package. | Rejected |

### Glob: `path/filepath.Glob` vs hand-rolled

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **`path/filepath.Match`/`Glob`** | Standard lib. POSIX semantics. Cross-platform. | No extended globs (`**`, `?(...)`). | **Chosen** |
| Hand-rolled / third-party | `**`, GNU extensions. | New dep, hand-rolled correctness risk. | Rejected |

### Command substitution: recursive `shell.Run` vs separate substituter

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **Recursive call back into `Shell.runExternal` (capturing stdout)** | Reuses the entire dispatch tier (built-ins, cache, plugin, exec) so `$(persona list)` works. | Crosses the parser→shell boundary cleanly via a `CmdSubExecutor` interface. | **Chosen** |
| Separate `expander.Substitute` that re-implements exec | No shell dep in expander. | Duplicates known-binary lookup + plugin dispatch — drift bait. | Rejected |

### Alias scope (this PR vs future)

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **First-token rewrite, 16-step recursion cap** | Matches bash/zsh "simple alias" behavior. Predictable. | Doesn't handle `alias rm='rm -i'` followed by `\rm foo` to bypass (escape-bypass) — deferred. | **Chosen** |
| Full bash-compatible alias (including escape bypass) | Bug-for-bug compat. | Significant tokenizer work. Not justified by current users. | Rejected |

## Testing strategy

- Parser-package unit tests for brace, glob, cmdsub (each isolated).
- Shell-package unit tests for each new built-in (`alias`, `source`,
  `set`, `unset`).
- Integration test in `shell_acceptance_test.go` that runs a synthetic
  REPL line through `dispatch` and asserts the alias rewrite.
- `make ci` from the worktree must exit 0.

---

## Risks

1. **Expansion order changes shadow existing behavior.** `runExternal`
   currently parses + execs the expanded line. We insert a
   post-parse expansion step that could surprise existing tests.
   Mitigation: brace/glob/cmdsub are no-ops on inputs without the
   trigger characters; existing tests use clean inputs.
2. **Cmdsub recursion can spawn unbounded children.** Mitigation:
   16-level depth cap with a clear error.
3. **Glob expanding to thousands of files.** Mitigation: rely on the
   underlying syscall (no extra cap this PR; matches bash default).

---

## Backward compatibility

- The new built-ins are additive — no existing dispatch arm changes.
- The expansion pass is additive — inputs without `{`, `*`, `?`, `[`,
  or `$(` flow through unchanged.
- The `[aliases]` RC table was already parsed (v0.3-1 #190); we now
  apply it. Users with that table set today gain functioning aliases.

---

## Out of scope (deliberate)

- Backtick command substitution.
- Extended globs (`**`, `?(...)`).
- `set -e`/`set -o` shell options.
- Bash alias escape-bypass (`\rm`).
- Nested braces (`{a,{b,c}}`).
