# Plan: v0.3-1 — Login Shell Capabilities

> Make aish a viable **login shell** on macOS and Linux. This PR delivers
> the narrow "login shell" surface (flag detection, RC sourcing,
> `logout`/`exec` built-ins, `/etc/shells` registration instructions);
> the broader job-control / brace-expansion / full-builtins surface
> tracked under #82 is **deferred** to follow-up epics with per-task
> stubs and a documented gap (per `Common.md P3 — Surface, Don't Bury`).

**Source:** GOALS.md §"Epic v0.3-1 — Login Shell Capabilities"
**Epic (supervised):** #82
**Tasks (in scope this PR):** #85 (login-shell registration), #86
(RC-file loading — file moved to `~/.aish/aishrc.toml`).
**Tasks deferred this PR:** #83 (fg/bg/jobs), #84 (process groups),
#87 (broad builtins — only `logout` and `exec` added; `cd`, `export`
already exist, `alias`/`source`/`set`/`unset` deferred), #88 (brace
expansion, glob, command substitution).
**Domain:** `domain:shell-runtime`
**Risk:** Medium per GOALS.md. The MVP boundary is drawn so the
shipped surface is *fully testable from Go* — RC sourcing is pure I/O
into `env.Env`; `logout`/`exec` are dispatch branches; `/etc/shells`
registration is a printed instruction, not a privileged write.
**Milestone:** v0.3 — Real Shell (90 days)

## Acceptance Criteria

### MVP-floor (shipping this PR)

- [ ] **Login-mode detection (#85 subset).** `aish` enters login mode
      when ANY of the following is true:
      1. `argv[0]` begins with a `-` (POSIX login convention — `getty`
         and `login` invoke shells this way).
      2. The `-l` flag is present.
      3. The `--login` flag is present.
      Detection happens in `shell/cmd/aish/main.go` before `shell.New()`.
      The `Shell` struct grows a `loginMode bool` field; the public
      constructor splits into `New()` (existing behavior, non-login)
      and `NewWithOptions(Options{Login: true})` so existing callers /
      tests are untouched.
- [ ] **RC sourcing order (#86).** When `loginMode == true`, `New*`
      sources two RC files in this order, **before** the cache /
      history / telemetry openers fire (so RC-set env vars like
      `ANTHROPIC_API_KEY` actually reach the plugin spawn):
      1. `/etc/aish/aishrc` (system-wide; missing file = silent skip)
      2. `$HOME/.aish/aishrc.toml` (per-user; missing file = silent skip)
      The user file overrides the system file on any key collision.
      Non-login mode does **not** source these files (POSIX
      convention — interactive non-login shells source a different
      RC, which we defer).
- [ ] **RC file format (TOML).** The RC file is TOML with three
      sections:
      ```toml
      [env]
      PATH = "/usr/local/bin:/usr/bin:/bin"
      AISH_THEME = "default"

      [shell]
      umask = "022"          # optional, octal string

      [aliases]              # accepted but not applied this PR
      ll = "ls -la"          # reserved for #87 follow-up
      ```
      The `[env]` table populates `env.Env` via `Set`. The `[shell]
      umask` field, when present, parses as an octal string and
      calls `syscall.Umask` on POSIX (no-op on Windows).
      `[aliases]` is **parsed and validated** but the aliases are
      not yet applied to dispatch — they live on the Shell as
      `s.aliases map[string]string` so #87 can wire them later.
      Parse errors (invalid TOML, non-string env value, malformed
      umask) write a one-line warning to stderr and the shell
      continues — RC failures must not deny login.
- [ ] **`$AISH_VERSION` propagation.** The shell sets
      `AISH_VERSION` in its env (overriding any inherited value) to
      the build-time `version` string from `main.go`. Set via a new
      `(*Shell).SetVersionString(string)` call before RC sourcing,
      so the RC's `[env]` block can reference it via TOML string
      interpolation in the future (this PR: read-only).
- [ ] **POSIX `PATH` default.** When `loginMode == true` AND
      `$PATH` is unset after RC sourcing, the shell sets
      `PATH=/usr/local/bin:/usr/bin:/bin` (Linux/macOS) or leaves it
      alone on Windows. Mimics bash's `--login` behavior.
- [ ] **`logout` built-in.** New dispatch branch in `shell.go`:
      - In login mode: clean shutdown — equivalent to EOF on stdin.
        Implementation: `logout` returns `io.EOF` up through `Run`
        so the existing Close path fires.
      - In non-login mode: prints `aish: logout: not a login shell`
        to stderr, exits with code 1 (matches bash).
      - Accepts an optional integer argument for the exit code
        (`logout 5` → process exits 5). Default 0.
- [ ] **`exec <cmd>` built-in.** New dispatch branch in `shell.go`:
      - With no args: no-op (POSIX behavior — applies redirections
        only, but we have none yet → silent success, exit 0).
      - With a command: parses + execs the command via
        `syscall.Exec` so the **aish process is replaced** by the
        child (POSIX semantics: `exec /bin/bash` becomes bash with
        the same PID). On Windows: `syscall.Exec` is unavailable;
        fall back to running the child and `os.Exit(child.ExitCode)`.
        Failure to resolve the binary on PATH writes
        `aish: exec: <cmd>: command not found` and exits 127 (the
        login shell exits — `exec` failure is fatal by POSIX).
- [ ] **`/etc/shells` registration instructions (#85).** New `make
      install-shells` target at the **top-level** Makefile that
      detects platform and prints (does NOT write) the instructions
      a user runs to register aish. macOS and Linux paths covered;
      Windows path prints "not applicable." Sample output:
      ```
      $ make install-shells
      → To register aish as a login shell:
        sudo sh -c 'echo /usr/local/bin/aish >> /etc/shells'
        chsh -s /usr/local/bin/aish
      ```
      The Makefile target itself does **no** privileged writes —
      `Common.md §2.2` forbids the assistant from `sudo`'ing without
      explicit confirmation; the target is **instruction-only**.
- [ ] **Sample RC file.** `data/aish/aishrc.example` — a documented
      TOML template a user can copy to `~/.aish/aishrc.toml`. Has
      one comment line per section explaining purpose.
- [ ] **`make ci` passes** from the worktree.
- [ ] **Live smoke:**
      1. `./shell/dist/aish -l` with a synthetic
         `~/.aish/aishrc.toml` setting `[env] FOO = "bar"`; then in
         the REPL `echo $FOO` prints `bar`.
      2. In a login session, `logout` exits cleanly.
      3. In a non-login session (`./shell/dist/aish` with no flag),
         `logout` prints "not a login shell" and `exit` (not yet a
         built-in — synthesized by EOF) terminates.
      4. `exec /bin/echo replaced` replaces the aish process with
         `echo`; output is `replaced` and the parent shell sees the
         echo exit code.

### Deferred (acknowledged stubs / follow-up issues)

- [ ] **#83 Job control (fg, bg, jobs).** Requires `setpgid`,
      `tcsetpgrp`, `SIGTSTP`/`SIGCONT` handling — invasive across
      `internal/exec`. **Deferred to v0.3-1b** with its own plan.
      The `Shell` struct gains a `jobs []job` field placeholder
      (empty struct now) so #83 can wire without rewriting.
- [ ] **#84 Process group management.** Same cluster as #83.
      **Deferred to v0.3-1b.** A `// TODO(#84)` marker is added at
      the point in `internal/exec` where we'd call `setpgid` on
      child spawn.
- [ ] **#87 Full builtin set.** `cd` and `export` already exist
      (v0.1-1). This PR adds **`logout`** and **`exec`** (the
      login-shell-specific pair). **`alias`**, **`source`**,
      **`set`**, **`unset`** are deferred to v0.3-1c. The `[aliases]`
      RC section is parsed today so `alias` can become a thin
      wrapper later without an RC migration.
- [ ] **#88 Brace expansion, glob, command substitution.**
      Significant parser work (every token grows a "needs-expansion"
      step; the parser package itself is partition-locked by another
      TL). **Deferred to v0.3-1d** as its own epic.

### Out of scope (deliberate exclusion, not deferral)

- Writing to `/etc/shells` or running `chsh` from the shell process.
  Per `Common.md §2.2` these are privileged + destructive — they
  remain user actions, not assistant actions.
- Sourcing fish/bash/zsh RC files. aish has its own RC; mimicking
  POSIX RC syntax invites bug-for-bug compat we don't want.
- `~/.aish/config.toml` migration. The existing v0.2-5 / v0.3-3
  `config.toml` (active theme, active persona, telemetry opt-in)
  stays untouched. `aishrc.toml` is a NEW file with a different
  contract (env / shell options / aliases). The two coexist.

## Rationale + Alternatives Tables

### Alternative: RC file format

| Alternative                       | Pros                                                                 | Cons                                                                  | Verdict |
|---|---|---|---|
| **TOML** (`aishrc.toml`)          | Matches existing aish config files (config.toml, identity.toml). Already a dep (`BurntSushi/toml v1.6.0`). Structured — no parser tricks.   | Not POSIX-shell-syntax-compatible; users can't copy `~/.bashrc` over. | **Chosen** |
| POSIX-shell syntax (`.aishrc`)    | Familiar to bash/zsh users; `export FOO=bar` works.                  | Requires a shell-syntax parser; quoting/eval rules are a rabbit hole. | Rejected |
| JSON                              | Standard-lib only; no new dep.                                       | No comments; verbose; doesn't match other aish config files.          | Rejected |
| YAML                              | Comments + nesting.                                                  | New dep; YAML's "anything is a string until it isn't" trap.            | Rejected |

### Alternative: System-RC location

| Alternative                       | Pros                                                                  | Cons                                                                | Verdict |
|---|---|---|---|
| **`/etc/aish/aishrc`**            | Matches bash (`/etc/bash.bashrc`), zsh (`/etc/zshrc`). Trivial for distro packagers. | Some BSD/legacy systems prefer `/usr/local/etc/` for non-base configs.   | **Chosen** |
| `/usr/local/etc/aish/aishrc`      | Homebrew-style; matches some macOS conventions.                       | Linux distros (the primary login-shell deployment target) expect `/etc/`. | Rejected |
| `/etc/aishrc` (no dir)            | One fewer mkdir.                                                       | No room for a future drop-in `/etc/aish/aishrc.d/` pattern.            | Rejected |
| `$XDG_CONFIG_DIRS/aish/aishrc`    | Standards-compliant.                                                   | Not what login-shell admins look for; bash/zsh don't honor this either. | Rejected |

### Alternative: How to be a login shell

| Alternative                                  | Pros                                                | Cons                                                              | Verdict |
|---|---|---|---|
| **Detect `-l`, `--login`, dash-argv[0]**     | Matches bash; works for `login(8)`, GDM, sshd.       | Three code paths converge.                                         | **Chosen** |
| Only `--login` flag                          | Single code path.                                    | Breaks `login(8)` / `chsh`-installed convention (dash-argv[0]).    | Rejected |
| Environment variable (`AISH_LOGIN=1`)        | No flag parsing.                                     | Not how login programs invoke shells; user-foot-guny.              | Rejected |

### Alternative: `logout` and `exec` placement

| Alternative                       | Pros                                          | Cons                                              | Verdict |
|---|---|---|---|
| **Dispatch branches in `shell.go`** | Matches `cd` / `export` placement; consistent. | Grows `dispatch()` by ~20 lines.                  | **Chosen** |
| Separate `builtin_logout.go` + `builtin_exec.go` | One file per built-in matches v0.3-3 pattern. | Adds two files for ~30 lines total.               | Also chosen (one file per built-in for parity with `builtin_identity.go` / `builtin_secret.go`). |

Resolution: Built-ins live in their own files (`builtin_logout.go`, `builtin_exec.go`), invoked from a one-line dispatch branch in `shell.go`. Matches v0.3-3 convention.

## Approach (numbered steps)

1. Add `Options` + `NewWithOptions` to `shell.go`; refactor
   existing `New()` to call `NewWithOptions(Options{Login: false})`.
   This keeps every existing test green.
2. Create `login.go`:
   - `(*Shell).setLoginMode(bool)`,
   - `loadRC(path string) error` — TOML decode into a local
     `rcFile` struct, apply `[env]` to `s.env`, parse `[shell]
     umask` and call `syscall.Umask` (POSIX) / no-op (Windows),
     store `[aliases]` on `s.aliases`.
   - `applyLoginEnvDefaults()` — sets `AISH_VERSION` and
     POSIX-default `PATH` when unset.
3. Create `login_test.go` — fixture RC files written into `t.TempDir()`,
   `HOME` pivoted via `t.Setenv` so the loader picks them up.
   Cases:
   - Missing both files → no error, no env change.
   - System-only file → its env is applied.
   - User-only file → its env is applied.
   - Both files, conflicting key → user wins.
   - Malformed TOML → warning to stderr, shell still constructs.
   - `[shell] umask = "077"` → `syscall.Umask` called (verify via
     post-call file create + stat).
4. Create `builtin_logout.go` — implements `logout [N]`. Returns a
   sentinel error (`ErrLogout` exported from the package) that
   `runStream` / `runTTY` recognize and treat as a clean shutdown
   like EOF. Non-login mode → exit 1 with the bash error string.
5. Create `builtin_exec.go` — implements `exec <cmd …>`. On POSIX,
   uses `syscall.Exec` after resolving via `osexec.LookPath` and
   building `argv` + `envv` from `s.env.Environ()`. On Windows,
   spawns the child, waits, calls `os.Exit(child.ExitCode())`.
   No args → exit 0 silently. Failure to resolve → exit 127.
6. Wire dispatch branches in `shell.go` for `logout` and `exec`.
7. Update `cmd/aish/main.go` to parse `-l` / `--login` and inspect
   `argv[0][0] == '-'`; pass `Options{Login: true, Version: version}`
   to `NewWithOptions`. Preserve the existing `--version` / `--help`
   short-circuits.
8. Write `data/aish/aishrc.example`.
9. Add `install-shells` top-level Makefile target.
10. Run `make -C shell ci` until clean.

## Testing strategy

**Unit** (Go test, `go test ./internal/shell/...`):

- `TestLoadRC_*` — 6 cases above.
- `TestApplyLoginEnvDefaults_AISHVersion` — `AISH_VERSION` is
  always overwritten to the build-time string, even if inherited.
- `TestApplyLoginEnvDefaults_PathDefault` — empty PATH gets POSIX
  default in login mode; non-empty PATH is left alone.
- `TestLogoutBuiltin_LoginMode` — returns `ErrLogout` (or equivalent
  sentinel propagated up through dispatch).
- `TestLogoutBuiltin_NonLoginMode` — exit 1, stderr message.
- `TestLogoutBuiltin_WithExitCode` — `logout 7` propagates 7.
- `TestExecBuiltin_NoArgs` — silent success.
- `TestExecBuiltin_NotFound` — exit 127, stderr message.
- `TestExecBuiltin_ResolvesBinary` — happy-path *up to but not
  including* `syscall.Exec` (we'd never return). Indirection via
  an injected `execFn func(...) error` so the test can assert the
  resolved binary path and argv.

**Integration** (Go test in `shell/integration/`):

- `TestLoginShell_DashArgv0` — spawn `aish` with `argv[0] = "-aish"`,
  pipe in `echo $AISH_VERSION; exit`, assert stdout matches the
  build-time version.
- `TestLoginShell_FlagL` — spawn with `-l`, same assertion.
- `TestLoginShell_RCFile` — write a temp RC, set `HOME`, spawn
  with `-l`, pipe `echo $FOO`, assert `bar`.

**Live smoke** (manual, scripted in the final report):

- The four bullets under "Live smoke" above.

## Risk assessment

| Risk                                                          | Mitigation                                                                                      |
|---|---|
| `syscall.Exec` differs on Windows                             | Build-tag split (`exec_posix.go`, `exec_windows.go`); Windows path is documented as a fallback. |
| RC parsing throws during cache/history init                   | RC is loaded **before** the openers in `NewWithOptions`; an error from RC writes stderr and is non-fatal. |
| `umask` is global per-process and leaks to children           | Documented behavior — POSIX shells work the same way; this is *the point* of `umask`.            |
| `logout` sentinel error confused with real I/O error          | Use a typed `errLogout struct{ code int }` distinct from `io.EOF`; only the dispatcher inspects it. |
| Existing tests break when `New()` changes shape               | Keep `New()` as a thin wrapper around `NewWithOptions(Options{Login:false})`.                    |
| `AISH_VERSION` overrides a user-set var                       | Explicitly documented: login-mode sets it; non-login leaves user's value alone.                  |

## Backward compatibility

- `New()` keeps its existing signature and behavior. Non-login
  invocations of aish are unchanged.
- The new RC file is **opt-in**: missing → silent skip. Users who
  never set `aish` as their login shell never see the new code path.
- `AISH_VERSION` is a new variable; collisions with user-set values
  in login mode are documented in the example RC.
