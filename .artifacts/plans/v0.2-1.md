# Plan: v0.2-1 — Interactive Shell UX

> Make aish feel genuinely good to use as a layer inside any terminal.
> Reach: replace the byte-by-byte stdin reader on the interactive path
> with a real TTY-aware line editor — history navigation, ghost-text
> suggestion, smart tab completion, fuzzy history search. The scripted
> (`cat | aish`) path keeps the issue-#167-safe byte-by-byte reader.

**Source:** GOALS.md §"Epic v0.2-1 — Interactive Shell UX"
**Epic (supervised):** #45
**Tasks (in-scope this PR):** #46, #47, #48, #49, #50, #51
**Domain:** `domain:ui`
**Risk:** LOW (per GOALS.md). Concentrated in one new package; the
existing byte-by-byte path stays intact for non-TTY stdin so issue
#167's regression seatbelt continues to hold.
**Milestone:** v0.2 — Layer Polish

## Acceptance Criteria

- [ ] **#46 BubbleTea-equivalent TUI integration.** Interactive REPL
      input flows through a TTY-aware line editor (raw mode + ANSI key
      decoding) when stdin is a terminal. Non-TTY stdin still uses the
      byte-by-byte `readLine` path so `cat scripted-input | aish` and
      the issue-#167 seatbelt continue to pass. *Note on framework:* we
      do not vendor BubbleTea itself in this PR — see the
      "Alternatives Table — line editor" below. The Epic's
      framework-name in GOALS.md ("BubbleTea TUI integration") is a
      stand-in for "a real interactive UX layer"; the deliverable is
      the UX, not the dependency.
- [ ] **#47 Ghost-text suggestions.** As the user types, a dim
      suffix is rendered after the cursor showing the most-recent
      history entry that starts with the current input. `Right` or
      `End` accepts the suggestion (full); `Ctrl-F` accepts one word.
      The suggestion is rendered with the active theme's
      `ghost_suggestion` role (currently muted). Empty input shows no
      suggestion. **Source:** session history only in this PR; cache
      suggestion suffix is deferred (see Alternatives Table).
- [ ] **#48 Syntax highlighting as-you-type.** First-token coloring
      based on resolution tier: known-binary green (`ai_tier_local`),
      cache/inference fallback blue (`ai_tier_cloud`), built-in
      yellow (`accent`). Quoted strings render in `string` role.
      Highlighting runs on every redraw; cost ≤ 50µs per redraw on a
      120-char buffer (test-asserted). Falls back to plain rendering
      when stdin is not a TTY.
- [ ] **#49 Smart tab completions.** `Tab` cycles through completions
      for the token under the cursor. First-token completion uses
      `$PATH` binaries + the aish built-ins (`cd`, `export`, `theme`,
      `cache`, `undo`, `restore`, `stats`). Subsequent tokens use
      filesystem path completion against the current working directory
      (including `~`-expansion). Multiple matches: `Tab` shows the
      common prefix and a one-line summary; a second `Tab` cycles. No
      completion for the AI-intent path in this PR (the line is opaque
      until inference). Git-ref completion is **out of scope** for
      this PR — see Out of Scope.
- [ ] **#50 `Ctrl-R` reverse history search.** Pressing `Ctrl-R`
      enters search mode: prompt becomes `(reverse-i-search)`; typing
      filters history substring-wise; `Ctrl-R` again advances to the
      next match; `Enter` accepts and dispatches; `Esc` cancels and
      restores the pre-search buffer. Fuzzy match (substring + case-
      insensitive) for this PR; FZF-style scoring is out of scope.
      History source is `~/.aish/history.db` (read-only) plus the
      current session's lines.
- [ ] **#51 Nerd Font prompt + segments.** The existing `theme.Active`
      already renders cwd / git / ai-tier / drachma segments. This
      task extends the editor so it RE-RENDERS the prompt on
      `SIGWINCH`, on theme change (the theme builtin already exists),
      and on every key — the prompt stays aligned when the line wraps.
      The Nerd Font glyphs themselves are theme data (already in
      `bundled.go`); this PR makes sure the editor doesn't clip them
      when redrawing.
- [ ] **`make ci` passes** with `GOFLAGS=-buildvcs=false` (worktree
      VCS-stamping is unrelated to this PR — see "Verification gates"
      in HANDOFF/Spawn notes).
- [ ] **Live smoke** in a real TTY:
      1. `make -C shell build && shell/dist/aish`
      2. Press `Up` after running `echo one` — line shows `echo one`.
      3. Type `ec` and see `echo one` as ghost-text suggestion.
      4. Press `Tab` after `cd /us` and see `/usr/` filesystem
         completion.
      5. Press `Ctrl-R`, type `ech`, see `echo one`.
      6. Press `Ctrl-C` mid-edit — terminal is restored to cooked
         mode; aish prints a fresh prompt; no orphan raw-mode state.

## Architecture

```
shell/internal/
├─ term/                              NEW package — TL_UI owns entirely
│  ├─ doc.go         Package doc: scope, non-TTY fallthrough, contract
│  │                   with shell.Run.
│  ├─ tty.go         IsTTY + RawMode (enter / restore via x/term).
│  ├─ tty_test.go    Verifies RawMode round-trips termios state.
│  ├─ keys.go        Key + KeyType + ANSI-CSI decoder. Pure-function
│  │                   transform from a stream of bytes to Key events.
│  ├─ keys_test.go   Table-driven decoder cases: arrows, Home, End,
│  │                   Ctrl-A/E/U/W/F/R/C/D, Backspace, Delete, Tab,
│  │                   Enter, plain runes, multi-byte UTF-8.
│  ├─ buffer.go      Line buffer: rune-indexed insert / delete /
│  │                   move-cursor / kill-word / kill-to-end-of-line.
│  │                   No I/O; pure state.
│  ├─ buffer_test.go
│  ├─ history.go     HistoryNav: walk up/down through a snapshot of
│  │                   prior commands; restores a stash on bottom.
│  ├─ history_test.go
│  ├─ complete.go    Completion engine: token-under-cursor resolver,
│  │                   filesystem completer, PATH-binary completer,
│  │                   built-in completer. CompletionSource interface.
│  ├─ complete_test.go
│  ├─ ghost.go       GhostSuggester: substring-prefix lookup over the
│  │                   history snapshot. Returns the suffix-after-buffer
│  │                   for the most-recent match, or "" when none.
│  ├─ ghost_test.go
│  ├─ search.go      ReverseSearch state machine: query buffer + match
│  │                   pointer; Step(key) → (newState, accepted bool).
│  ├─ search_test.go
│  ├─ render.go      Renderer: writes the current frame (prompt + line
│  │                   + ghost + cursor) to an io.Writer using ANSI
│  │                   movement. Idempotent: every key triggers a full
│  │                   redraw of the current row(s).
│  ├─ render_test.go
│  ├─ highlight.go   SyntaxColor(line, resolver) → []Span. The resolver
│  │                   tells the renderer whether the first token is a
│  │                   known binary, built-in, or AI-intent; spans
│  │                   carry a theme role name (NOT a concrete color).
│  ├─ highlight_test.go
│  ├─ editor.go      Editor: the loop that owns one input session.
│  │                   ReadLine(ctx) (string, error) drives raw-mode,
│  │                   decodes keys, mutates buffer, redraws, returns
│  │                   on Enter / Ctrl-D / Ctrl-C.
│  └─ editor_test.go Uses a fake terminal (in-memory pipe pair) to
│                      exercise the loop without a real TTY.
└─ shell/
   ├─ shell.go              EDIT (surgical, ≤30 lines added):
   │                          - Detect TTY at Run() entry.
   │                          - When TTY: construct term.Editor,
   │                            override the per-iteration read with
   │                            editor.ReadLine.
   │                          - When NOT a TTY: keep the existing
   │                            readLine byte-by-byte loop verbatim.
   │                          - Prompt() unchanged; ReadLine takes the
   │                            prompt string as input.
   │                          - History snapshot built from the
   │                            history.Store (read-only) at Run()
   │                            entry; updated in-session by appending
   │                            each accepted line.
   └─ shell_test.go          Tests for the non-TTY path remain green.
```

## Alternatives Table — line editor implementation

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **A. Vendor `golang.org/x/term` for raw mode + write our own editor in `shell/internal/term/`** | Single small new dep (golang.org/x/term — Go-blessed, BSD-3, already a transitive dep via go-isatty's siblings). Full control over key handling, ghost-text composition, highlighting integration, the Ctrl-R UI. Aligns with `Code.md §6` (justify deps; the alternative is a lot of dep). Aligns with the project pillar "OS-insensitive" — we own the cross-platform termios surface. | We write more code (~1.2k LoC across the term package). Have to handle Windows console mode separately (deferred via build tags). | **CHOSEN.** |
| **B. Vendor `github.com/chzyer/readline`** | Mature, battle-tested. Includes history, completion, vi/emacs modes out of the box. BSD-3 license. | Adds a third-party transitive dep with active maintenance concerns (last meaningful release was years ago — fails `Code.md §6` "no release in 24 months" smell). Its rendering model fights ours (we need theme-aware coloring tied to `shell/internal/theme`). Its completion API is opinionated and would need a shim. We'd be vendoring two-thirds of what we don't need. | Rejected. |
| **C. Vendor `github.com/charmbracelet/bubbletea` and write a Bubbles-based editor** | The literal text of the epic in GOALS.md says "BubbleTea TUI integration." First-class component ecosystem (textinput, viewport). Bright future. | TUI-style full-screen rendering does not match aish's pillar of "any terminal, line-edit only." BubbleTea owns the whole screen by design — a shell that takes over the terminal between commands is not a layer, it's a takeover. Performance overhead (re-render whole screen per keystroke) is wrong for ghost-text + a 120-char buffer. Heavy dep (~30k LoC across bubbletea + bubbles + lipgloss). | Rejected for v0.2-1. The epic title is aspirational; the right read is "interactive UX layer," which option A delivers. If a future surface (e.g. `aish history` viewer) demands a full-screen TUI, BubbleTea is the right tool *there*, not in the input loop. |
| **D. Pure stdlib (no x/term).** | Zero new deps. | Termios manipulation in pure Go is portable in theory but ugly in practice — manual syscall.Termios, raw `tcgetattr`/`tcsetattr` over `unix.IoctlGetTermios`. The `golang.org/x/term` package is the standard wrapper everyone uses and exists for exactly this. | Rejected. |

**Choice: A.** Lightest dep footprint, best fit with existing theme
package, no fight against the "any terminal" pillar.

## Alternatives Table — ghost-text suggestion source

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **A. History only.** Use `history.Store.List(N)` for the last N lines and pick the most-recent prefix-match. | Zero coupling to cache. Predictable. Works offline. Matches how fish and zsh-autosuggestions ship by default. | Misses the AI-native angle: a cached intent ("delete logs older than 30 days") wouldn't surface as a suggestion until *after* the user has run it once. | **CHOSEN for v0.2-1.** |
| **B. Cache-first.** Query `cache.Store` for intents whose embedding prefix matches the current input. | Surfaces AI-native suggestions before the user has typed them once. The community-cache (#L3) hypothesis suggests this would feel magical on day one. | Cache lookup on every keypress is too slow for a tight ghost-text loop (10ms p50 per the cache plan is OK *once*, fatal at 60Hz). Would need a debounce + worker. Embedding similarity is the wrong primitive for prefix-matching — we'd need an FTS index over `intents`. | Rejected for v0.2-1; revisit when the cache package adds an FTS index (out-of-scope for this Epic). |
| **C. Both, history wins ties.** | Best of both. | Combines the cons of A and B. The composition layer becomes the bug surface. | Rejected — not before B's debouncing problem is solved. |

**Choice: A.** History-only ghost-text is the v0.2-1 deliverable; cache-suggestion ghost-text is a v0.2-2 candidate filed as a TODO comment.

## Test Strategy

Per `Code.md §3` (testing pyramid):

**Unit (the bulk):** every file in `shell/internal/term/` has a peer
`_test.go`. The interesting cases:

- `keys_test.go` — table-driven: every key sequence we recognize,
  including the multi-byte CSI prefixes (`\x1b[A`, `\x1b[1~`, …) and
  the partial-sequence case (timer-driven decode of a bare `\x1b` as
  ESC).
- `buffer_test.go` — invariant: every operation preserves the rune
  index; cursor never falls outside `[0, len(runes)]`.
- `complete_test.go` — golden-file path completions against a
  `t.TempDir()` filesystem; PATH-binary completions against a stubbed
  `$PATH` directory; built-in completer hard-coded.
- `editor_test.go` — drives the full Editor.ReadLine via an in-memory
  pipe: writes bytes to one end, reads frames from the other, asserts
  the final accepted line. Covers: Enter, Ctrl-C (no line, terminal
  restored), Ctrl-D (EOF on empty buffer), Up/Down history, Tab
  completion cycle, Ctrl-R search.
- `tty_test.go` — `IsTTY(io.Reader)` returns false for `bytes.Buffer`;
  RawMode against a `pty.Open` allocation round-trips termios.
  Skipped on Windows for now.

**Integration:** the existing `shell/integration/` suite continues to
pass unchanged because integration tests pipe scripted stdin (non-TTY)
and hit the byte-by-byte path. We add ONE new integration test that
verifies non-TTY input still works — a regression seatbelt for the
issue-#167 fix when the editor lands.

**Adversarial pass (per task §6 of the spawn workflow):**
- Pipe non-TTY input through `cat | aish` — must hit byte-by-byte.
- Press Ctrl-C mid-edit — terminal must be restored.
- Arrow keys, Home/End, Ctrl-A/E/U/W must be idempotent at buffer
  boundaries (no panic, no clamp-to-negative).
- Tab must not consume a key when ghost-text is showing — Tab cycles
  completions, Right accepts ghost-text. Distinct keys, distinct
  semantics.

**Performance budget (acceptance criteria, asserted in tests):**
- A redraw on a 120-char buffer with highlighting MUST be < 1ms on
  the test machine. Asserted via `go test -bench` table in
  `highlight_test.go`; failing budget is a test failure.

## Out of scope (explicit)

- **Full vi/emacs binding parity.** We ship a fixed emacs-style
  keymap: Ctrl-A/E/U/W/F/B + arrows + Home/End/Backspace/Delete +
  Tab/Enter/Ctrl-C/Ctrl-D/Ctrl-R. No vi-mode toggle. (`Code.md §1`
  intent-revealing names — when the user requests vi-mode we'll add
  a keymap registry; for now the simpler thing is right.)
- **Multi-line editing.** The buffer is single-line. Wrapping is the
  terminal's job; line continuation (`\` at EOL) is not a v0.2-1
  feature.
- **Git-ref completion (`git checkout <ref><Tab>`)** — listed in
  Epic #45's brief but requires a git-aware completer that knows how
  to read `.git/refs`. Defer to a future v0.2-1.1 PR.
- **FZF-grade fuzzy scoring for Ctrl-R.** Substring + case-insensitive
  is the v0.2-1 bar; we'll port the FZF v2 algorithm later if the
  feedback is "it feels slow."
- **Windows console raw-mode.** `tty.go` has a build-tagged file pair;
  Windows ships with the byte-by-byte fallback in v0.2-1. Filed as a
  follow-up against the v1.0 Windows-native epic.
- **Cache-driven ghost suggestions.** Per the second Alternatives
  Table. TODO comment in `ghost.go`.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Terminal left in raw mode after panic / SIGSEGV / Ctrl-C | `defer term.Restore(...)` at every Editor.ReadLine entry; signal handler installs a restore-and-rethrow for SIGINT/SIGTERM. Test in `editor_test.go` asserts the deferred restore runs on every exit path. |
| Issue-#167 regression — TTY editor steals bytes that should have flowed to a child | The TTY path is only entered when `IsTTY(stdin)` is true. The scripted-stdin path (`cat | aish`) hits the existing byte-by-byte `readLine`. Existing integration test `TestCatConsumesPipedStdin` is the seatbelt; we add a second one for symmetry. |
| Ghost-text steals Tab | Tab is bound to completion, NOT ghost-acceptance. Ghost-acceptance is Right / End / Ctrl-F. Test in `editor_test.go` exercises both. |
| Theme role lookup at 60Hz is slow | Highlight passes a `theme.Active` snapshot once per ReadLine entry; per-keystroke render only reads pre-resolved color bytes. `highlight_test.go` asserts the per-redraw budget. |
| `golang.org/x/term` adds a Windows-conditional surface I have to maintain | Build-tag the Windows fallback as a TODO. The shell still runs on Windows (byte-by-byte path) until a Windows-specific raw-mode is added. |

## Dependencies

- **New Go dependency:** `golang.org/x/term v0.20.0+` (BSD-3,
  Go-blessed, transitively present already via `golang.org/x/sys`).
  Justified in the Alternatives Table.
- **Existing internal dependencies (read-only):**
  - `shell/internal/theme` for the active theme + role resolution.
    `term/highlight.go` and `term/render.go` accept a `*theme.Theme`
    (the snapshot returned by `themes.Active()`) so they don't import
    the registry.
  - `shell/internal/history` for the read-only `Store.List(N)` call to
    build the history snapshot. Read-only — we do NOT add to the
    history schema in this PR.
  - `shell/internal/env` for `$PATH` resolution during binary
    completion.
- **Blocks downstream:** none in v0.2-1. Theme work (TL_THEME) and
  PTY work (TL_PTY) are orthogonal.

## Backward compatibility

- **Non-TTY behavior is unchanged.** Same `readLine`, same exit
  semantics, same `$?` propagation, same issue-#167 guarantee.
- **TTY behavior is new.** Users running `aish` interactively see
  the new editor by default. No flag toggles it off in v0.2-1; if a
  user reports breakage, `AISH_NO_LINE_EDITOR=1` will be added in a
  hot-fix.
- **History on-disk schema unchanged.** We read via the existing
  `Store.List` API; no migrations.
