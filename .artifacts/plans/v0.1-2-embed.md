## Objective

Make the v0.1-2 intent cache hit on *semantically equivalent* intents — not
only on byte-identical normalized strings — by storing a vector embedding
alongside each row and routing cache misses through a cosine-similarity
lookup before falling back to the inference plugin.

## Acceptance Criteria

- [ ] **#16** `aish-inference-cloud` answers `proto.MethodEmbed` requests by POSTing to `<base>/embeddings` and returning a `Vector []float32` + `Cost` in a `proto.Frame` of `Kind=embedding`. Default base URL = `https://api.convergent-systems.co/llm/v1` (the same gateway path commit `7c243bd` set for `/messages`).
- [ ] **#16** `cache.PluginClient.Embed(ctx, text) ([]float32, error)` sends a `MethodEmbed` request and returns the vector.
- [ ] **#16** Embedding cost is recorded distinct from infer cost (model id passes through the same `proto.Cost` plumbing so the existing `reliab.Cost` recorder accumulates a separate model bucket).
- [ ] **#17** `Store.Write(intent, os, invocation, confidence, embedding)` persists the embedding as little-endian float32 bytes in the `embedding BLOB` column already reserved in DDL.
- [ ] **#17** `Store.LookupNearest(embedding, threshold, os) (intent, invocation, similarity, hit, err)` returns the *single highest-similarity* row for the given `os` whose cosine similarity to the query vector is at or above `threshold`. Returns `hit=false` when no row meets threshold or when no row has an embedding.
- [ ] **#17** `Cache.Resolve` order: exact-hash match → embedding-similarity lookup (only if PluginClient supports Embed) → plugin Infer → write-back with the freshly-generated embedding so the next call lands an exact-hash hit.
- [ ] **#17** Per-OS isolation preserved: `LookupNearest` MUST NOT consider rows for a different OS.
- [ ] **#17** Threshold is configurable. Default `0.85` (initial flywheel calibration; tighten in v0.2 once we have telemetry).
- [ ] **No regression on `d01388d`:** every existing test in `shell/internal/cache/` and `plugins/cloud/...` still passes; the exact-hash path remains zero-RPC.
- [ ] `make ci` exits 0.

## Wire-shape decision (the one boundary issue worth surfacing)

The existing `proto.Request.Params` field is typed as `InferParams`, and
the existing `rpc.Dispatcher` invokes handlers with `proto.InferParams`.
The user spec says "Append-only — DO NOT change existing types."

**Resolution:** Embed requests reuse the existing `InferParams` envelope —
the `Intent` field carries the text to embed, the `Model` field carries
the embedding model id, and the rest of the fields stay at zero-value.
This is semantically clean (an intent IS the text we're embedding), and
it requires no change to `Request`, `InferParams`, `rpc.Dispatcher`, or
`rpc.Handler`. `EmbedParams` and `EmbedResult` are introduced as
top-level helper types for *callers* — handlers map from `InferParams`
to `EmbedParams` in one line at the dispatch boundary.

The Frame carries the vector via an additional optional field. This is
the one place we must touch existing types: `Frame` is appended with
`Vector []float32 \`json:"vector,omitempty"\``. Append-only and
backwards-wire-compatible (existing consumers ignoring unknown fields
keep parsing). The new `KindEmbedding = "embedding"` Kind constant is
added — also append-only.

## Alternatives Table — similarity metric

| Alternative                              | Pros                                                                                          | Cons                                                                                                                  | Verdict          |
|------------------------------------------|-----------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------|------------------|
| **Cosine similarity**                    | Direction-only; scale-invariant. Standard for sentence/text embeddings. Range -1..1 → easy threshold UX. | Requires per-vector L2 norm at compare time (cheap but non-zero).                                                     | **Chosen**       |
| Dot product (un-normalized)              | Fastest (one multiply-accumulate per dim, no sqrt). Works if vectors are L2-normalized.       | Sensitive to vector magnitude — gateway-side norm contract is not documented, so we'd be coupling to an upstream detail. | Rejected (today) |
| Euclidean distance                       | Geometrically intuitive; popular in vector DBs.                                               | For unit-norm text embeddings cosine and Euclidean are monotonically related, but Euclidean's threshold doesn't transfer across embedding-model dim/scale changes. Cosine is the portable choice. | Rejected         |

If gateway-side L2 normalization is later contractually guaranteed, switching
cosine → dot-product is a one-line change in `similarity.go`; isolating it
behind `similarity.Cosine(a,b)` keeps the swap cheap.

## Alternatives Table — vector storage layout

| Alternative                                              | Pros                                                                                                                | Cons                                                                                                            | Verdict       |
|----------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------|---------------|
| **BLOB of little-endian float32 bytes in `intents.embedding`** | Schema already provisioned (`embedding BLOB`, DDL line 45). Zero migration. Idempotent. Brute-force scan acceptable at v0.1 corpus sizes (~thousands of rows). | Full table scan per lookup. For 100K+ rows we'd want an ANN index — out of scope for v0.1.                      | **Chosen**    |
| Separate `vectors` table with FK to `intents`            | Cleaner separation of concerns; lets a future ANN index live in its own table without bloating the hot row.         | Requires DDL migration. Two-table read on every lookup. Premature factoring for a v0.1 corpus.                  | Rejected (today) |

If row count crosses ~50K we revisit with a sqlite-vss extension or a
SQLite-resident ANN; that's a v0.2-3 concern per GOALS.md L2/L3 cache.

## Test Strategy

### Unit tests (must pass under `go test -race -count=1`)
- **proto:** round-trip `EmbedParams` + a Frame with `Type=embedding` and a populated `Vector`; assert JSON tags and `omitempty` for the new field on non-embedding frames.
- **anthropic.Client.Embed:** success (200 + JSON vector payload), 401 → `CodeAuthFailed`, 429 → `CodeRateLimited`, 5xx → `CodeInternal`, ctx-deadline → `CodeTimeout`, no API-key leak in any error string, no API-key leak in `Client.String()`.
- **Store.Write w/ embedding:** byte-equal round-trip of float32 LE bytes; `LookupNearest` after Write returns the row at similarity 1.0 (self-similarity).
- **Store.LookupNearest threshold:** seed three rows with distinct embeddings; assert highest match is returned at threshold 0.0, only above-threshold matches are returned at 0.99, none returned at 1.01.
- **Store.LookupNearest per-OS isolation:** seed identical embeddings under different OS keys; query for one OS, assert the other-OS row is invisible.
- **Cache.Resolve precedence:** exact-hash hit takes priority over similarity (close the plugin after pre-populating an exact match — Resolve still returns the cached invocation without RPC, and `fromCache=true`).
- **Cache.Resolve similarity branch:** seed a row whose embedding is the stub-plugin's deterministic vector for a paraphrased intent; new call with the paraphrase hits the similarity branch, returns the stored invocation, `fromCache=true`, no Infer RPC issued (assert via stub-call-counter).
- **Cache.Resolve threshold respected:** identical setup but query whose stub-Embed vector lands below threshold falls through to Infer.

### Integration / plugin
- **Plugin Embed end-to-end:** httptest stub returning a JSON `{"embeddings":[{"embedding":[…]}]}` body; assert the plugin's MethodEmbed reply has the same vector + a populated `Cost` with model id distinct from any infer model.
- **Stub-plugin MethodEmbed branch:** the in-repo test stub emits a deterministic vector derived from `hash(intent)` so per-OS / threshold tests can reason about the result without floating-point noise.

### CI gate
- `make ci` exits 0 on the merge SHA.

## Risks + mitigations

| Risk                                                                                       | Mitigation                                                                                                                   |
|--------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------|
| `api.convergent-systems.co/llm/v1/embeddings` DNS not yet live from dev host               | httptest stub covers all unit tests; runtime path returns a typed error (`CodeInternal`/transport) until DNS lands — explicitly noted as v0.1 limit. |
| Embedding model id drifts between vendor calls, breaking cosine across the cache           | Persist the model id alongside each row in a follow-up; for v0.1, an embedding mismatch falls through to Infer (correct but slow). |
| BLOB-LE codec endian footgun on big-endian hosts                                           | We pin LE in the codec (`binary.LittleEndian`) and round-trip-byte-equal it in tests. Go's only realistic targets are LE; we still pin explicitly. |
| Similarity branch silently regresses exact-hash performance                                | `Resolve` calls `LookupNearest` ONLY after exact-hash miss — codified in the precedence test (`exact_takes_priority`).        |
| Threshold poorly calibrated → false-positive cache hits return wrong invocation            | Default 0.85 is conservative; in v0.2 we will collect telemetry on similarity-hit user-acceptance and tighten/loosen.         |
