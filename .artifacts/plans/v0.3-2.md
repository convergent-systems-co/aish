# Plan: v0.3-2 — Plugin Registry

> Generalise the hard-coded `aish-inference-cloud` spawn into a **signed
> local-first plugin registry**. A plugin is a binary that speaks the
> existing `libs/proto/inference` NDJSON wire shape; the registry is N
> manifest files at `~/.aish/plugins/<name>/manifest.json` that aish
> reads at startup. Trust mirrors v0.2-3 community-cache: Ed25519
> signatures over the binary digest, anchors compiled in. A new
> `plugin` built-in administers the registry. Backward compat: when
> the registry is empty, fall back to the existing PATH lookup so
> every existing user keeps working without intervention.

**Source:** GOALS.md §"Epic v0.3-2 — Plugin Registry"
**Epic (supervised):** #89
**Tasks (in scope this PR):** #90, #91, #92, #93, #94
**Domain:** `domain:plugin`
**Risk:** MEDIUM (per GOALS.md). The trust + lifecycle surface is the
load-bearing piece. The plan partitions trust verification into a tiny
subpackage (`libs/proto/registry/`) reused by both the loader and the
admin CLI, so the shell's hot path (Resolve → PluginClient) only gains
one additional manifest-lookup step.
**Milestone:** v0.3 — Real Shell

## Acceptance Criteria

- [ ] **#90 JSON-RPC contract finalized.** The existing
      `libs/proto/inference` types (Request / Response / Frame / Kind)
      are documented as **the** plugin contract. A new
      `libs/proto/registry` package adds the manifest schema, the
      `Kind` capability advertisement, and the signature verification
      types. No breaking changes to `inference`; the registry is
      additive metadata about the binary.
- [ ] **#91 `plugin` built-in.** The shell exposes
      `plugin list | install <path> | remove <name> | verify <name>`
      from a new `shell/internal/shell/builtin_plugin.go`. List is
      read-only; install/remove/verify mutate the registry under
      `~/.aish/plugins/`. Concurrent installs are serialised on a
      file lock.
- [ ] **#92 Local manifest format.** `manifest.json` per plugin with
      `format_version`, `name`, `version`, `binary_path`,
      `kinds` (e.g. `["inference"]`), `sha256` (hex digest of the
      binary), `signer_id`, `signature` (base64 over `sha256_bytes`).
      Format is JSON to match the existing community-cache manifest
      pattern; pinned by Go types in `libs/proto/registry/manifest.go`.
- [ ] **#93 First community plugin (Ollama scaffold).** The Ollama
      inference plugin is **scaffolded** as a manifest stub +
      documentation entry under `data/plugins/ollama/`; the full
      binary lands in a follow-up. The registry MUST accept the
      manifest shape an Ollama plugin would publish (kinds=inference,
      binary_path absolute, signer_id pinned). The existing cloud
      plugin gets a registry manifest as the canonical example.
- [ ] **#94 Lifecycle management.** Existing `cache.PluginClient`
      handles spawn + read-loop already. The registry loader picks
      the right binary based on the requested kind; the shell still
      owns one PluginClient per session. Crash isolation: if the
      child exits, the demultiplexer closes pending channels and the
      shell surfaces the error (existing behavior). A new
      `plugin restart` subcommand is **out of scope** for this PR —
      lifecycle here means "the registry knows how to spawn the right
      thing, and the shell can tear it down cleanly."
- [ ] **Backward compat.** If `~/.aish/plugins/` is empty or absent,
      the existing PATH lookup for `aish-inference-cloud` (or
      `$AISH_INFERENCE_PLUGIN` override) still works. Test-asserted.
- [ ] **Trust enforcement.** A manifest signed by an unknown signer
      or with an invalid signature is **rejected at install time**
      and never loaded at startup. The bundled cloud plugin's
      manifest is signed by the development anchor (same key as the
      community-cache dev signer — keeps the dev surface tiny). A
      dedicated `AISH_PLUGIN_TRUST_DISABLE=1` env var is **NOT**
      offered; trust is non-negotiable per `Code.md §7`.
- [ ] **`make ci` passes** from the worktree.
- [ ] **`make build`** produces `shell/dist/aish` +
      `plugins/cloud/dist/aish-inference-cloud` +
      `plugins/cloud/dist/aish-plugin`.

## Alternatives Table — Manifest Format

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **JSON (chosen)** | Same shape as v0.2-3 community-cache manifest. Standard library only. Human-readable, diff-friendly. No new schema language to learn. | Verbose vs TOML/YAML. Comment-unfriendly. | **Chosen** |
| TOML | Already in tree (config.toml). Comments. | Two formats (TOML for config, JSON for manifests) drifts further from each other; the *manifest* is structurally identical to community-cache's, and the community-cache code path is the reference. | Rejected |
| YAML | Most expressive. | New dep (`gopkg.in/yaml.v3`) for one file format. Indentation footguns. | Rejected |

## Approach

1. **libs/proto/registry** — types-only first. Manifest JSON shape + Validate() + Kind enum. Re-export `community.DevSigningSeed` via a thin shim so we don't duplicate the dev key.
2. **Trust + verify** — port the community-cache verify.go pattern: `HashBinary(path) → hex`, `SignBinary(priv, path) → base64`, `VerifyManifest(m, binaryPath) → error`.
3. **Registry loader** — scan `~/.aish/plugins/<name>/manifest.json`, validate each, return `[]Entry`. Skip invalid entries with a stderr warning (matches community-cache failure mode).
4. **Registry install** — write manifest under `~/.aish/plugins/<name>/manifest.json` after VerifyManifest passes. Use a `flock`-style lock file at `~/.aish/plugins/.lock` to serialise concurrent installs.
5. **Plugin selector** — `shell/internal/plugins/select.go` exposes `Select(kind, dotAish, env) (binPath string, ok bool)`. The cache wires this in at Start time.
6. **`aish-plugin` CLI** — minimal cobra-free CLI (`flag.NewFlagSet` per subcommand, same style as the existing cloud plugin's flag parsing).
7. **`plugin` built-in** — mirrors `builtin_community.go`. Subcommand dispatch + per-subcommand handler.
8. **Bundled cloud manifest** — `data/plugins/cloud/manifest.json` signed by the dev anchor via a `make plugin-manifest` recipe (same shape as `make bundle`).

## Testing strategy

- **`libs/proto/registry`** — table-driven tests for Manifest.Validate (every malformed field → ErrManifestMalformed); roundtrip Sign → Verify; tampered-binary → ErrSignatureInvalid; unknown signer → ErrUnknownSigner; revoked signer → ErrRevokedSigner.
- **Registry loader** — TestMain builds a fake plugin binary into a tempdir (same pattern as `cache/testdata/stub-plugin`), signs a manifest with the dev key via `registry.SignBinary`, drops it under `<tempdir>/.aish/plugins/fake/manifest.json`, and asserts `Load()` returns one entry. A malformed-manifest neighbour is silently skipped with a stderr warning.
- **Installer** — install + remove roundtrip on tempdir. Concurrent install via two goroutines: assert serialisation via the lock file (no torn writes).
- **`aish-plugin` CLI** — TestMain compiles the binary into a tempdir, then golden-output tests for `list`, `install --signer aish-dev`, `verify`, `remove`.
- **`shell/internal/plugins`** — selector test: empty registry → PATH fallback; populated registry → returns the manifest's binary_path; manifest with unknown kind → no match.
- **Shell built-in** — golden tests for `plugin list`, `plugin install <path>`, `plugin remove`, `plugin verify`. `dispatchTier` test ensures `plugin` is recognised as TierBuiltin.
- **Backward compat** — TestNoRegistryFallback: with `~/.aish/plugins/` absent, the shell still starts the cloud plugin via the existing PATH path.

## Risk assessment

| Risk | Mitigation |
|---|---|
| Trust anchor compromise allows arbitrary plugin install | Compiled-in anchors only; rotation requires a release. No env-var override. |
| `~/.aish/plugins/` is world-writable on shared systems | Manifests are signed; an attacker who writes a manifest cannot forge a valid signature. The binary digest is bound by the signature. |
| Race in concurrent install | File lock at `~/.aish/plugins/.lock`; held for the duration of the verify-and-write window. |
| Backward compat regression | Explicit TestNoRegistryFallback. The shell's `tryStartPlugin` keeps the PATH branch as the default fallback. |
| Plugin lists a `binary_path` that doesn't exist | Selector returns ok=false; the shell falls through to PATH. `verify` subcommand reports it. |

- Depends on v0.2-3 community-cache trust pattern (already merged) — we reuse the dev anchor concept.

## Backward compatibility

- Empty `~/.aish/plugins/` → existing PATH-based flow continues to work. No user-visible change.
- The shell's `cache.PluginConfig.BinaryPath` field is unchanged; the selector is only consulted when BinaryPath is empty AND the registry is non-empty.
- The `aish-inference-cloud` binary is unchanged — it remains the canonical inference plugin. Only its location is now optionally declared via a manifest.

---

## Out of scope

- **Plugin sandboxing** (seccomp / AppArmor / Job objects) — defer to v0.4.
- **Plugin marketplace** (browse, search, rate) — defer to v0.4 + ecosystem.
- **Capability negotiation beyond declared `kinds`** — the plugin advertises kinds in its manifest; runtime capability handshake is deferred.
- **`plugin restart` / health check** — the existing PluginClient already surfaces crash via channel close. A dedicated health-check subcommand is a follow-up.
- **Ollama plugin binary** — scaffolded only; the full Go binary is a separate PR.
- **Plugin lifecycle hot-reload** — restarting the child without restarting aish is deferred.
- **`plugin build ./local`** — the listed CLI surface in GOALS lists `build`, but the *build* step (`go build`) belongs to the plugin author's workflow, not to aish. We expose `install <path>` instead; the author runs `go build` themselves. Documented in the built-in's help.
