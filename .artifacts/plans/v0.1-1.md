## Acceptance Criteria

Each sub-issue's tick-box from GOALS.md becomes an explicit gate.

- [ ] **#2** `go.mod` + build pipeline (macOS + Linux). `go build ./cmd/aish` and a `Makefile` target produce a working binary on darwin/arm64; cross-compile commands documented.
- [ ] **#3** `aish` CLI launches an interactive prompt (REPL loop reading from stdin).
- [ ] **#4** Tokenizer handles quotes, flags, pipes. Round-trips through unit tests.
- [ ] **#5** External binaries execute via `os/exec` with stdin/stdout/stderr wired to the calling shell.
- [ ] **#6** Stdin/stdout/stderr pipes between two commands: `echo foo | tr a-z A-Z` works.
- [ ] **#7** `cd` and pwd tracking — working dir persists across commands.
- [ ] **#8** Environment variables: `export X=1; echo $X` works; child processes inherit env.
- [ ] **#9** Exit code captured in `$?`.
- [ ] **#10** Output type detection — `text`, `json`, `ndjson` classification probing first 512 bytes.
- [ ] **#11** Basic prompt rendered (`~/path > `) — no Nerd Font yet.

The v0.1-1 PR is **done** when `go build ./...` exits 0, `go test ./...` exits 0, and a smoke session in the REPL exercises the golden path (`cd`, `echo` piped to `tr`, exit-code capture).

## Sub-tasks (parallel-safe after seed lands)

Three coder partitions. Each owns a disjoint set of `shell/internal/` packages plus its own tests. None overlap on files outside `shell/cmd/aish/main.go` (which the TL touches only on the seed).

### T1 — Parser + Exec
**OWNS:** `shell/internal/parser/**`, `shell/internal/exec/**`
**DEPENDS_ON:** seed
**Sub-issues:** #4 (parser), #5 (exec), #6 (piping)
**ACCEPTANCE:**

- `parser.Parse(input string) (Pipeline, error)` returns a structured pipeline (commands + args + redirections). Handles single/double quotes, simple flags (`-x`, `--flag`, `--flag=value`), and `|`. No subshells, no backticks, no globbing yet.
- `exec.Run(ctx, pipeline, env, stdin, stdout, stderr) (exitCode int, err error)` runs the pipeline via `os/exec`. For `cmd1 | cmd2`, wires `cmd1.Stdout` to `cmd2.Stdin` via `io.Pipe` (or `exec.Cmd.StdoutPipe()`). Returns the exit code of the **last** command in the pipeline (POSIX semantic).
- Tests cover: simple invocation, two-stage pipe, missing binary (`exec: "frobnitz": executable file not found in $PATH`), arg with spaces inside quotes.

### T2 — Shell state (cwd + env + exit code + prompt)
**OWNS:** `shell/internal/shell/**` (except shell.go scaffold from seed), `shell/internal/env/**`
**DEPENDS_ON:** seed
**Sub-issues:** #3 (REPL), #7 (cwd), #8 (env), #9 (`$?`), #11 (prompt)
**ACCEPTANCE:**

- `shell.Shell` struct holds: `cwd`, `env []string`, `lastExit int`.
- `Shell.Run(stdin, stdout, stderr)` enters a REPL: render prompt → read line → if `cd <path>` then chdir; if `export NAME=VALUE` then set env; else parser+exec.
- `$?` is expanded by a pre-parse substitution pass (simplest: regex replace `$?` and `${?}` with the string of `lastExit`).
- `$VAR` and `${VAR}` substitution sourced from `Shell.env`.
- Prompt format: `<cwd-shortened> > ` where `~` substitutes for `$HOME` prefix. No git segment, no Nerd Font.
- Tests cover: `cd /tmp` updates cwd; `export FOO=bar; echo $FOO` prints `bar`; running a failing command sets `$?` to non-zero.

### T3 — Output stream type detection
**OWNS:** `shell/internal/stream/**`
**DEPENDS_ON:** seed
**Sub-issues:** #10
**ACCEPTANCE:**

- `stream.Detect(b []byte) stream.Kind` probes up to 512 bytes and returns `KindText | KindJSON | KindNDJSON | KindTable`.
- JSON: starts with `{` or `[`, parses fully as `json.RawMessage`.
- NDJSON: every non-empty line parses as JSON.
- Table: tab-separated, ≥2 columns, ≥2 rows.
- Fallback: `KindText`.
- Tests cover: each kind with positive examples; a confusing input like `not json` returns text; ambiguous JSON-looking text without close-brace returns text (not JSON).

## Test Strategy

- **Unit:** every `shell/internal/*` package ships with its own `*_test.go`. Table-driven where useful.
- **Integration:** one `shell/internal/shell/integration_test.go` boots a `Shell` and feeds it a scripted stdin (multi-command session). Asserts on stdout + final exit code. **No mocks for exec** — the test calls real `echo`, `tr`, `cat`. If a test box lacks them, the test should `t.Skip` with a clear reason.
- **Adversarial focus areas** (for the tester):
  - Coverage theater: any `if err != nil { return }` swallowing must be checked for a test that exercises the error path.
  - `try/except: pass` analog — `defer recover(); _ = recover()` shorthand without explicit decisions.
  - `TODO` / `FIXME` / `XXX` in committed Go files.
  - Skipped tests (`t.Skip`) with no environmental reason.
  - Type assertions and `interface{}` use without surrounding type-switch handling.
