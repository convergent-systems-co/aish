## Acceptance Criteria

- [ ] **#23** JSON-RPC plugin contract published at `libs/proto/inference/` (Request, Response, Token, Complete types). NDJSON line-oriented over stdin/stdout. Versioned (`jsonrpc: "2.0"`). Documented in package godoc.
- [ ] **#24** `plugins/cloud/cmd/aish-inference-cloud/main.go` builds a self-contained binary. Reads JSON-RPC requests on stdin, dispatches `infer` to the Claude API client, writes NDJSON responses on stdout.
- [ ] **#25** Streaming NDJSON: each token from the Anthropic stream becomes one `{"jsonrpc":"2.0","id":..,"result":{"type":"token","data":"<chunk>"}}` line. Terminal `complete` line carries the full invocation + confidence.
- [ ] **#26** Configurable per-request timeout (default 30s); 3 retries with exponential backoff for retryable HTTP statuses (429, 5xx). Non-retryable errors (4xx other than 429) surface immediately.
- [ ] **#27** API key read from `$ANTHROPIC_API_KEY`. Plugin fails fast at startup if unset, with a clean error line. Key NEVER appears in stdout, stderr, logs, or any captured trace (per `Common.md §4`).
- [ ] **#28** Per-request cost line emitted to a configurable log (default `~/.aish/cost-log.jsonl`). One JSONL row per request: `{"chronon":"<RFC3339>","model":"<name>","tokens_in":N,"tokens_out":M,"usd":F,"req_id":"<uuid>"}`.

The v0.1-3 PR is **done** when `make ci` from the monorepo root exits 0 (incl. integration tests against a stubbed Anthropic endpoint) and the binary builds for darwin/linux × amd64/arm64 + windows × amd64/arm64.

## Sub-tasks (parallel-safe after seed)

Three coder partitions with disjoint `OWNS:` directories. None overlap on files outside the seed.

### T1 — Plugin core (CLI + JSON-RPC dispatcher)

**OWNS:** `plugins/cloud/cmd/aish-inference-cloud/main.go` (extend), `plugins/cloud/internal/rpc/**`
**DEPENDS_ON:** seed
**Tasks closed:** #24 (partial — plugin binary boundary)
**ACCEPTANCE:**

- `main()` reads `$ANTHROPIC_API_KEY`, fails with non-zero exit + clean stderr if unset (per #27 seed sets this stub; T1 fills it). The actual API call wiring is T2 — T1 only dispatches.
- `rpc.Dispatcher` reads NDJSON requests from `io.Reader`, validates each as a `proto.Request`, calls a registered handler, writes the result (or `{"jsonrpc":"2.0","id":..,"error":{...}}`) as NDJSON to `io.Writer`.
- Handler registration: `Register("infer", func(ctx, params) <-chan proto.Frame)`. The dispatcher streams frames as they arrive.
- Tests: dispatcher round-trips one synthetic request through a fake handler; dispatcher returns error frame for malformed JSON; dispatcher honors context cancellation.

### T2 — Anthropic API client + streaming NDJSON

**OWNS:** `plugins/cloud/internal/anthropic/**`
**DEPENDS_ON:** seed
**Tasks closed:** #24 (the Anthropic side), #25, #27
**ACCEPTANCE:**

- `anthropic.Client` constructed with `(apiKey, baseURL, httpClient)`. `apiKey` is held privately; `client.String()` MUST NOT emit it.
- `client.Infer(ctx, params) (<-chan proto.Frame, error)` opens an SSE-style streaming POST to `/v1/messages` with `stream: true`. Each delta from the model becomes a `proto.Token` frame; the terminal `message_stop` becomes a `proto.Complete` frame carrying the assembled `invocation` text and the response's reported `confidence` (or `1.0` if Anthropic doesn't expose it for the model).
- Streaming tests run against `httptest.Server` returning canned SSE bytes — NO real Anthropic in CI.
- One opt-in smoke test reads `$ANTHROPIC_API_KEY_INTEGRATION` (different name to avoid CI confusion) and hits `api.anthropic.com`; `t.Skip` if unset.
- API key NEVER logged or returned in error messages — must be redacted to `[REDACTED:api-key]` if it would ever surface.

### T3 — Reliability + cost tracking

**OWNS:** `plugins/cloud/internal/reliab/**`
**DEPENDS_ON:** seed, T2's `proto.Frame` consumption shape (no file overlap — T3 wraps T2's `Infer` rather than modifying it)
**Tasks closed:** #26, #28
**ACCEPTANCE:**

- `reliab.WithRetries(ctx, fn, opts) (T, error)` wraps a `func() (T, error)` with 3 attempts, exponential backoff (250ms, 500ms, 1s), and a retry predicate (default: HTTP 429 or 5xx).
- `reliab.WithTimeout(ctx, duration) (context.Context, context.CancelFunc)` — straightforward but tested for goroutine leakage.
- `reliab.Cost` records per-request tokens + estimated USD (pricing table embedded as a `var` keyed by model id), appends a JSONL line to a configurable writer. Default writer = file at `~/.aish/cost-log.jsonl`; dependency-injected for tests.
- Tests: retry triggers on 429, gives up after 3 attempts; cost-log row matches schema exactly; cost-log file location respects `$HOME` / `$USERPROFILE` per the same `homeDir` semantics as the shell module (helper duplicated locally OR imported from `shell` — TL decides; defer to T3's coder).

## Test Strategy

- **Unit (per package):** every `internal/<pkg>/` has its own `*_test.go` with table-driven tests. Race-detector mandatory.
- **Integration (`plugins/cloud/integration/`):** spawns the built plugin binary as a subprocess; feeds JSON-RPC requests over stdin; reads NDJSON responses; asserts on frame sequences. Anthropic endpoint mocked via `httptest.Server` started by the harness — the binary is told the base URL via `--api-url` flag (T2 owns this flag).
- **NO real Anthropic in CI.** The opt-in smoke test (in T2 acceptance) is gated on `$ANTHROPIC_API_KEY_INTEGRATION` and `t.Skip`s without it. CI never has that var.
- **Adversarial focus areas** (for the tester):
  - **Secret leakage:** grep all `*.go` for `apiKey`, `Authorization`, `Bearer` — every appearance MUST be in a path that redacts or writes only to authenticated HTTP, never to logs/stdout/stderr/errors.
  - **Swallowed errors** in retry path (`for i := 0; i < N; i++` patterns).
  - **Goroutine leaks** in streaming reader (use `goleak.VerifyTestMain` or equivalent).
  - **Cost arithmetic** — off-by-one on token counts, FP comparison without tolerance.
  - **Pricing table drift** — flag a TODO if the constants drift from a documented source.

## Risks

| Risk | Mitigation |
|---|---|
| Anthropic SSE format drifts | Canned-fixture tests are the only authority for the parser. If real API changes, smoke-test catches it; CI is decoupled. |
| API key leaks to stderr via panic stack trace | Add a top-level recover in `main()` that redacts and re-raises; tester scans for "Bearer " / "sk-" patterns in any test capture. |
| Goroutine leak on context cancel of streaming reader | Use `goleak` in T2 tests + ensure the SSE reader's goroutine selects on `ctx.Done()`. |
| Cost-tracking file grows unbounded | Out-of-scope for v0.1-3; v0.1-5 telemetry epic handles rotation. Note in T3 godoc. |
| Cross-platform path semantics for cost log | T3 uses the `homeDir(env)` pattern from PR #170 so `$HOME` / `$USERPROFILE` both work. |
