# Plan: v0.2-api-fix — Switch cloud plugin to the CS gateway wire shape

> Bug-fix wave for the v0.2 milestone. The cloud inference plugin
> currently speaks Anthropic Messages API to a path the gateway does not
> serve. This plan migrates the plugin to the OpenAI chat-completions
> shape on Cloudflare Workers AI that the gateway actually implements
> (per `core-infra/terraform/cloudflare/workers-ai/src/worker.js` and
> `core-infra/workers/auth-proxy/src/index.js`), gated behind the
> bearer-token auth proxy at `api.convergent-systems.co/v1/*`.

**Source:** Issue #178 (`[bug] plugins/cloud speaks wrong wire protocol to gateway`)
**Predecessor commits:** `7c243bd` (PR #174 — origin of the wrong base URL), `9351289` (v0.1-5 telemetry merge — current main)
**Domain:** `domain:inference`
**Risk:** HIGH (release-blocker for v0.2-3 community cache and v0.2-4 script translation)
**Milestone:** v0.2 — Layer Polish (60 days)

## Objective

Make `aish-inference-cloud` actually reach the gateway:
URL = `https://api.convergent-systems.co/v1/chat/completions`,
auth = `Authorization: Bearer <token>`,
body = OpenAI chat-completions,
SSE = OpenAI delta chunks terminated by `[DONE]`.
Embeddings are not yet served by the gateway (tracked in
`core-infra#10`); the embed path returns a typed sentinel and the
cache's similarity branch becomes a no-op until the gateway endpoint
ships.

## Acceptance Criteria

- [ ] **#178** Package `plugins/cloud/internal/anthropic` is renamed to
      `plugins/cloud/internal/csllm` via `git mv` so import paths track
      the gateway's actual identity (Convergent Systems LLM gateway) and
      not the misleading "anthropic" name.
- [ ] **#178** Default base URL is `https://api.convergent-systems.co/v1`;
      the infer path appends `/chat/completions` (not `/messages`).
- [ ] **#178** Request body is OpenAI chat-completions shape:
      `{model, messages:[{role,content}], stream, max_tokens?, temperature?}`.
- [ ] **#178** Auth is `Authorization: Bearer <token>`. The legacy
      `x-api-key` + `anthropic-version: 2023-06-01` headers are removed
      from every outbound request.
- [ ] **#178** SSE parser consumes OpenAI delta frames
      (`data: {"choices":[{"delta":{"content":"x"}}]}`), tolerates empty
      deltas and finish-reason-only frames, and terminates on the literal
      sentinel `data: [DONE]`.
- [ ] **#178** Default model is `@cf/meta/llama-3.1-8b-instruct`. The
      allowlist mirrors the worker:
      `@cf/meta/llama-3.1-8b-instruct`, `@cf/meta/llama-3.1-70b-instruct`,
      `@cf/meta/llama-3-8b-instruct`. Unknown-model pricing is 0.0 USD
      (non-fatal, same as today's Anthropic-pricing branch).
- [ ] **#178** Embedding requests return a typed
      `proto.ErrEmbedNotImplemented` sentinel. The auth-proxy /
      workers-ai worker exposes no `/embeddings` endpoint at the time of
      this fix; sending one would 404 and surface as `CodeInternal` —
      the sentinel makes the unsupported-feature path explicit.
- [ ] **#178** `shell/internal/cache.PluginClient.Embed` propagates the
      sentinel verbatim. `cache.Cache.Resolve`, when it observes the
      sentinel, skips the similarity branch and proceeds straight to
      `Infer` (no degradation, no `write-back without embedding` regression
      beyond what v0.1-2 already supports — `Store.Write(..., nil)` is
      the documented no-embedding path).
- [ ] **#178** `ANTHROPIC_API_KEY` and `ANTHROPIC_BASE_URL` remain
      accepted for backward compatibility with the v0.1 cohort; the new
      `CS_API_KEY` and `CS_BASE_URL` names take precedence when both are
      set, and a one-line deprecation warning fires on the legacy names.
- [ ] Four anti-regression grep guards return empty:
      - `grep -r '"/messages"' plugins/cloud/internal/`
      - `grep -r 'x-api-key' plugins/cloud/internal/`
      - `grep -ri 'anthropic-version' plugins/cloud/internal/`
      - `grep -r '/llm/v1' plugins/cloud/`
- [ ] `make ci` from the `aish-api` worktree exits 0.
- [ ] httptest stubs in `csllm_test.go`, `sse_test.go`, and
      `integration/plugin_test.go` produce wire-identical payloads to
      the real gateway (path = `/chat/completions`, header =
      `Authorization: Bearer <token>`, body = OpenAI shape, SSE =
      OpenAI deltas + `[DONE]`).
- [ ] Live smoke (manual, post-merge): mint a token via
      `core-infra/scripts/cs-token issue aish`, export as
      `CS_API_KEY` (or back-compat `ANTHROPIC_API_KEY`), run
      `aish-inference-cloud` against a `ping` then a streamed `infer`
      request.

## Alternatives — env-var naming

The plugin's existing env vars (`ANTHROPIC_API_KEY`, `ANTHROPIC_BASE_URL`)
are factually wrong: there is no Anthropic in the call path. But the
v0.1 cohort has these names baked into their shells, the v0.1-5
telemetry plan documents them, and the bug is "wire protocol," not
"env-var naming." Three options:

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| Keep `ANTHROPIC_API_KEY` only | Zero churn for existing users; PR is purely the wire fix. | The name lies about what it is. Future contributors will be confused — the bug itself was partly caused by mis-labelling. | Rejected — perpetuates the original mistake. |
| Introduce `CS_API_KEY` only, breaking change | Honest naming; clean break. | Every v0.1 shell breaks at next launch. The fix becomes a config migration, expanding blast radius beyond #178. | Rejected — scope creep, gratuitous user pain. |
| Support both, prefer `CS_*`, deprecation warning on legacy | Honest going forward; back-compat through v0.2; gives ops a published deprecation window. | Doubles the env-var surface temporarily. One extra branch in main.go. | **Chosen** — surfaces honesty without breaking the cohort. |

## Alternatives — embeddings-disable mechanism

The gateway does not implement `/embeddings` (per
`workers-ai/src/worker.js` — only `/v1/chat/completions` is served).
`Cache.Resolve` must skip the similarity branch without breaking the
v0.1-2 contract. Three options:

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| Typed `proto.ErrEmbedNotImplemented` sentinel | Smallest possible proto change (append-only); pattern matches existing `errors.Is` discipline in `cache.Resolve`; surfaces honestly through the JSON-RPC `Error.Code` channel. | Adds one constant + one var to `libs/proto/inference/inference.go`. | **Chosen** — minimal, typed, fits existing error plumbing. |
| Feature-flag method on `PluginClient` (`SupportsEmbed() bool`) | No proto change. | A capability flag is wrong when the plugin *could* embed but the gateway has not deployed the endpoint yet — the gating belongs at the wire boundary, not the client. Requires every plugin to grow a capability table. | Rejected — wrong layer. |
| New `CodeNotImplemented` proto error code, return at handler | Generic, reusable for future not-implemented surfaces. | Two changes (code + plumbing) where one (sentinel) suffices for the one call site that cares today. We add `CodeNotImplemented` *and* the sentinel: the code generalizes for future use; the sentinel is what `cache.Resolve` switches on. | **Augments the chosen alternative** — append both the code and the sentinel, both are append-only and cheap. |

```
│  ├─ embed.go                # Client.Embed: returns ErrEmbedNotImplemented

│  └─ embed_test.go           # Asserts the sentinel + zero HTTP calls.

   └─ main.go                 # Import path updated; env-var doc updated;

                              # deprecation warning on the legacy names.

                              # skip similarity branch and go straight to Infer.
```

## Test Strategy

httptest stubs match the real gateway wire shape exactly. Each test
asserts one wire property in isolation so a future drift surfaces as a
single targeted failure, not a generic "request rejected" assertion:

- `TestInfer_PostsToChatCompletionsPath` — path is exactly `/chat/completions`.
- `TestInfer_SendsBearerAuth` — header is `Authorization: Bearer <token>`.
- `TestInfer_OmitsXAPIKeyHeader` — `x-api-key` is absent.
- `TestInfer_OmitsAnthropicVersionHeader` — `anthropic-version` is absent.
- `TestInfer_BodyIsOpenAIShape` — JSON marshal pins
  `{model,messages:[{role,content}],stream:true}` (snapshot via
  `json.Unmarshal` + map-equality so field-order changes are tolerated).
- `TestInfer_SSE_AssemblesDeltas` — three `data: {...delta:content...}`
  lines + `data: [DONE]` produce three Token frames + one Complete.
- `TestInfer_SSE_EmptyDeltasTolerated` — `{"choices":[{"delta":{}}]}`
  and `{"choices":[{"finish_reason":"stop"}]}` neither crash nor emit
  Token frames.
- `TestInfer_SSE_DoneTerminator` — Complete frame fires on `[DONE]`,
  even without a preceding finish-reason chunk.
- `TestInfer_401_AuthFailed`, `TestInfer_429_RateLimited`,
  `TestInfer_500_Internal` — coded errors preserved with the new
  wire path (these survive from the renamed package's existing suite).
- `TestEmbed_ReturnsErrEmbedNotImplemented` — sentinel returned without
  hitting the network; an `httptest.Server` that fails the test if
  called confirms zero requests.
- `TestCache_ResolveSkipsSimilarityOnSentinel` — Resolve falls through
  to Infer cleanly when Embed returns the sentinel; v0.1-2 exact-hash
  branch still works.

Opt-in live smoke: `TestInfer_RealGateway_SmokeTest_OptIn` reads
`CS_API_KEY_INTEGRATION` (renamed from `ANTHROPIC_API_KEY_INTEGRATION`)
and hits the production gateway with `@cf/meta/llama-3.1-8b-instruct`.
CI never sets `CS_API_KEY_INTEGRATION`; this test skips in CI.

## Risk Assessment

| Risk | Mitigation |
|---|---|
| Renaming the package breaks v0.1-2 / v0.1-3 / v0.1-5 callers. | `git mv` preserves file history; import paths are updated atomically in one commit (`refactor(plugin): rename anthropic → csllm`); `make ci` from worktree root verifies. |
| Existing tests pass against the OLD wire shape (false-positive coverage). | TDD step: every wire-shape assertion in the test suite is rewritten BEFORE the implementation moves, then the suite must fail; only after the rewrite does the implementation commit land. |
| Cache.Resolve degrades on plugin sentinel error (regression in v0.1-2's similarity branch). | The similarity branch was already best-effort (per `cache.go:131` — `embErr == nil` guard). The new `errors.Is(err, ErrEmbedNotImplemented)` arm is a TYPED skip — the rest of the branch's failure semantics are unchanged. Test `TestCache_ResolveSkipsSimilarityOnSentinel` pins this. |
| Live smoke test accidentally bills production. | `CS_API_KEY_INTEGRATION` is never set in CI; the test reads it explicitly and `t.Skip`s on absence. This pattern survives unchanged from the v0.1-3 `ANTHROPIC_API_KEY_INTEGRATION` discipline. |
| 5xx retries via `reliab.WithRetries` still work. | The retry wrapper is untouched; `HTTPStatusError.IsRetryable` already classifies 429 + 5xx. Tester adversarial pass verifies the retry path still fires under the new transport. |
