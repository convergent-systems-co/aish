# Plan: v0.2-4 — Script Translation

> Let aish read existing bash/zsh/fish scripts and (a) execute them
> natively via the normal dispatch tier, (b) explain them in plain
> language, and (c) migrate them to aish-native form. MVP: cover the
> ~80% of "ordinary" scripts; anything past that becomes a clearly-
> labelled `Unknown` AST node, surfaced — never silently dropped.

**Source:** GOALS.md §"Epic v0.2-4 — Script Translation"
**Epic (supervised):** #64
**Tasks (in-scope this PR):** #65, #66, #67, #68, #69, #70
**Domain:** `domain:os-translation`
**Risk:** MEDIUM (per GOALS.md). Concentrated in one new package
(`shell/internal/translate/`) plus one new built-in file. The existing
dispatch path is untouched except for an additive switch branch.
**Milestone:** v0.2 — Layer Polish

## Acceptance Criteria

- [ ] **#65 bash reader.** `translate.ReadBash(src string) (*Script, error)`
      parses a bash script into the shared AST. Covers the 80% surface
      (see Scope Contract below). Comments are preserved as `Comment`
      nodes. Constructs the parser doesn't recognise become `Unknown`
      nodes with the original source line attached — they do NOT abort
      the parse.
- [ ] **#66 zsh reader.** `translate.ReadZsh(src string) (*Script, error)`.
      For MVP, zsh ≈ bash. The reader is a thin shim that calls into the
      bash reader plus the few zsh-specific lexical adjustments we
      handle (mainly: zsh comments inside `[[ … ]]`, `=(...)` process
      substitution flagged as `Unknown`, `setopt`/`unsetopt` treated as
      regular commands).
- [ ] **#67 fish reader.** `translate.ReadFish(src string) (*Script, error)`.
      Fish syntax is materially different (`set var value`, `if … end`,
      `for x in … end`, no `$()` — uses `(…)`). A separate lexer covers
      the same conceptual surface and emits the same AST.
- [ ] **#68 `aish run <script>`.** New built-in. Detects dialect by
      shebang first, file extension second, content heuristic third.
      Parses → walks the AST → executes each statement through the
      existing dispatch tier (`s.runExternal` for commands, in-process
      for control flow). Comments are no-ops at run time. Unknown nodes
      print `aish: run: line N: cannot translate (<reason>)` to stderr
      and abort with exit 2 — half-running is forbidden. Exit code is
      that of the last executed statement.
- [ ] **#69 `aish explain <script>`.** Deterministic AST-walk produces
      a numbered-step plain-language description. The output is the
      same every time for the same input. With `--with-llm` AND an
      `ANTHROPIC_API_KEY` set, an additional paragraph from the
      inference plugin is appended; without either, no LLM call is
      made.
- [ ] **#70 `aish migrate <script>`.** Walks the AST and emits an
      aish-native script that, when run via `aish run`, produces the
      same observable behavior. Comments are preserved verbatim.
      Unknown nodes become `# aish: MIGRATE-TODO: <reason> — <original
      line>` in the output so the user sees what didn't translate.
- [ ] **`make ci` passes** with `GOFLAGS=-buildvcs=false` (worktree
      VCS-stamping is unrelated, per PR #180/#183).
- [ ] **Live smoke.** A sample bash script using `echo`, `if/then/fi`,
      `for/do/done`, `|`, `>` runs cleanly under `aish run`;
      `aish explain` produces a sensible numbered description;
      `aish migrate` emits a valid aish-native script.

## Alternatives Table — Parser strategy

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| Hand-rolled tokenizer + recursive-descent parser | Pure Go, zero deps, matches `shell/internal/parser` style, scope-bounded to the 80% list, fast to compile | Bash edge cases (`[[ ]]`, here-strings, parameter expansion modifiers, arrays) need careful `Unknown`-classification — we'd have to write that bookkeeping ourselves | **Chosen** |
| Vendor `mvdan.cc/sh/v3` (battle-tested bash parser) | Full POSIX + bash + mksh coverage; well-maintained | Adds ~300KB to the binary, drags in BSD-3 third-party code we'd need GoLicense review for, and the AST shape doesn't match what we need to emit aish-native | Rejected — over-spec for the MVP |
| WASM tree-sitter grammars | Multi-dialect coverage, reusable | Adds a WASM runtime, opaque-to-Go binary blobs, slow startup, off-budget complexity | Rejected — over-engineered |

The hand-rolled path matches the existing `shell/internal/parser`
philosophy ("simple-and-durable beats clever-and-fragile" — Common.md
P1) and keeps the dependency footprint flat.

## Alternatives Table — Explain output shape

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| Numbered-step list (one line per AST node, nested for blocks) | Deterministic, scannable, trivially diff-able in tests | Less "narrative" feel | **Chosen** |
| Free-form prose paragraphs | More natural for end users | Hard to make deterministic without an LLM; defeats the offline-default goal | Rejected for the deterministic baseline (LLM enrichment can still add prose when `--with-llm` is on) |
| Annotated AST tree (indented) | Most faithful to the source structure | Reads like an internals dump; not user-friendly | Rejected |

## Scope Contract — the 80% list

**Covered by MVP (must work for all three dialects unless noted):**

- Comments (`# …` for bash/zsh; `# …` for fish too)
- Simple command + args
- Pipes (`a | b | c`)
- Output redirect (`> file`, `>> file`)
- Input redirect (`< file`)
- Single + double-quoted strings (no escape inside single; basic `\"` inside double)
- Variable assignment (`FOO=bar`, no `export` syntax for assignment — `export FOO=bar` is a command)
- Variable expansion (`$VAR`, `${VAR}`) — passed through; aish's env layer handles it
- Command substitution `$(...)` — emitted as a nested `Command` node; backticks deferred
- `if … then … [elif …] [else …] fi`
- `for VAR in WORDS; do … done`
- `while CMD; do … done`
- `case WORD in PAT) … ;; … esac` (basic glob patterns; no `;&` / `;;&` fallthrough)
- Simple function definitions: `name() { … }` (bash/zsh) and `function name; …; end` (fish)
- fish: `set [-l|-g|-x] VAR VALUE`, `if … end`, `for X in … end`, `while … end`, `function … end`

**Deferred (parser MUST emit `Unknown` with reason; downstream surfaces it):**

- Here-documents (`<<EOF … EOF`) and here-strings (`<<<`)
- `[[ … ]]` extended tests (parsed as a single `Unknown` command)
- Arithmetic `((…))`, `$((…))`
- Arrays + associative arrays (`arr=(a b c)`, `${arr[@]}`)
- Process substitution `<(…)`, `>(…)`, zsh `=(…)`
- Parameter-expansion modifiers (`${var//pat/repl}`, `${var:-default}`, `${#var}` — the bare `$VAR` form is fine; the modifiers are deferred)
- ZSH globbing extensions (`**/`, qualifiers)
- Backtick command substitution (use `$(…)` — the dialect docs all recommend it anyway)
- `select`, `trap`, `exec` redirect-to-fd
- fish: `set -e`, `set -q`, `set -l -x` combined flags beyond the basics, `string` builtin with all modifiers
- Job control (`&`, `wait`, `fg`, `bg`)
- `&&` / `||` short-circuit (covered as `Unknown` for MVP — common but the AST shape is a separate node type; filed as follow-up)

## Test Strategy

1. **Unit tests** per reader: golden tests where each input file in
   `testdata/` parses to a known AST shape. Round-trip is not required
   for the reader alone.
2. **Explain golden tests.** Each testdata script has an
   `<name>.explain.golden` file checked into the repo. Test reads the
   script, runs `Explain`, compares to the golden. Determinism check.
3. **Migrate golden tests.** Same pattern with `<name>.migrate.golden`.
   The migrated script MUST be parseable by aish (the migrate test
   also runs `parser.Parse` on each emitted line).
4. **Run engine tests.** Use a fake `Runner` (no real exec) that
   records each statement passed to it. Asserts the AST walker visits
   nodes in the right order, applies the right substitutions for env,
   etc.
5. **Round-trip smoke** (gated `// +build smoke`): for a curated few
   scripts, `migrate(bash)` then `aish run` produces the same stdout as
   `bash` running the original. Off by default because it requires
   bash on PATH; runs in CI on Linux/macOS.
6. **LLM enrichment skip-when-off.** The explain-with-llm test calls
   `t.Skip()` when `ANTHROPIC_API_KEY` is unset — no mocked-LLM tests
   pass when the gate is off.

## Risk Assessment

- **Parser scope creep.** Bash is genuinely a swamp. Mitigation: the
  Scope Contract is a fence. Anything outside it becomes `Unknown`;
  we never half-implement. Tests assert that.
- **Determinism of explain.** Map iteration order in Go is randomised.
  Mitigation: the AST walk is order-preserving and uses no map
  iteration in the prose generator.
- **`aish run` leaking env between invocations.** Each `aish run`
  builds a fresh env copy from the shell's current env; the script's
  assignments mutate that copy, not the shell's. Asserted by test.
- **Detection misfires** on scripts without a shebang. Mitigation:
  detection cascade — shebang > extension > content heuristic (presence
  of `function … end` ⇒ fish; presence of `fi`/`done`/`esac` ⇒ bash);
  on tie, default to bash and surface the assumption in the output.

## Dependencies

- None new. All work uses the existing Go stdlib plus the in-tree
  `internal/cache.PluginClient` (read-only, behind an interface).

---

## Backward Compatibility

- New built-ins (`run`, `explain`, `migrate`) are net-new commands.
  Nothing pre-existing uses those names as bare-word built-ins; the
  shell will dispatch them ahead of the cache/PATH lookup, so a user
  with `/usr/local/bin/migrate` on PATH would see the aish built-in
  win — flagged in the PR body so the next maintainer knows.
- No changes to existing AST, parser, exec, or cache APIs.

---

## Out of Scope (deliberate; deferred to follow-up issues)

- LLM-driven migrate (rule-based only for MVP).
- Round-trip equivalence proof beyond the smoke set.
- `&&` / `||` short-circuit AST nodes (currently `Unknown`).
- Heredocs, arrays, parameter expansion modifiers (see Scope Contract).
- Plugin telemetry on `aish explain --with-llm` calls — telemetry
  package is partition-locked.
- Concurrent / parallel script execution (no `&` background).
- Persistent script-cache memoization — each `aish run` is a fresh
  invocation. Each invocation is a clean session; env vars do NOT leak
  between calls.
