## Acceptance Criteria

- [ ] **#13** Schema with `(intent_hash, os, intent, invocation, confidence, hit_count, timestamps)` PK = `(intent_hash, os)`. Embedding column reserved but NULL for v0.1-2.
- [ ] **#14** SQLite store at `~/.aish/cache.db` via `modernc.org/sqlite` (pure-Go, no CGO).
- [ ] **#15** `Lookup(intent, os)` returns `(invocation, true)` on exact-match (intent hash) hit; `("", false)` on miss. Hit increments `hit_count` and updates `last_used`.
- [ ] **#18** `Write(intent, os, invocation, confidence)` upserts. Idempotent.
- [ ] **#19** Per-OS resolution: same intent can have different invocations for darwin / linux / windows; each lives in its own row.
- [ ] **#20** Stats: `Hits / Misses / TotalQueries` counters persisted in a `stats` k/v table.
- [ ] **#21** `aish cache stats` prints `Hits: N | Misses: M | Hit rate: P% | Entries: K`. `aish cache clear` truncates.
- [ ] **REPL tier order (GOALS.md):** built-in → known-binary passthrough → cache hit → inference plugin → write back.
- [ ] **Plugin client:** spawn `aish-inference-cloud` (path from `$AISH_INFERENCE_PLUGIN` env, default to PATH lookup), keep alive across requests, JSON-RPC over stdin/stdout.

## Architecture

```
shell/internal/
├─ cache/
│  ├─ schema.go    DDL + migrations
│  ├─ store.go     SQLite Store: Open/Close/Lookup/Write/Stats/Clear
│  ├─ plugin.go    PluginClient: spawn child, send JSON-RPC, receive NDJSON
│  └─ cache.go     Cache: orchestrates Store + PluginClient (Resolve method)
└─ shell/
   ├─ shell.go         dispatch: built-in → known-binary → cache → plugin
   └─ builtin_cache.go aish cache stats / clear
```

## Test Strategy

- **Unit per package** with table-driven tests over SQLite under tempdir, plugin spawn against a stub binary.
- **Integration**: spawn aish-inference-cloud as a child, end-to-end intent → infer → cache → second-call-hit.
- **No real Anthropic** in CI — plugin's `--api-url` already supports httptest stub.

## Out of scope (deferred)
