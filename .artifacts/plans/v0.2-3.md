# Plan: v0.2-3 ŌĆö Community Cache (L3)

> Layer an L3 community-curated cache **on top of** the v0.1-2 L1 local
> cache so a fresh install starts warm. The L3 bundle is a **read-only,
> Ed25519-signed SQLite file** distributed alongside the aish release. On
> first run aish verifies the bundle's signature against compiled-in
> trust anchors, copies it under `~/.aish/community-bundle.db`, and
> consults it on every L1 miss before invoking the inference plugin. On
> an L3 hit, the answer is **promoted to L1** with `source=community`
> provenance so the next call lands on the existing exact-hash path.

**Source:** GOALS.md ┬¦"Epic v0.2-3 ŌĆö Community Cache (L3)"
**Epic (supervised):** #58
**Tasks (in scope this PR):** #59, #60, #61, #62, #63

## Acceptance Criteria

- [ ] **#59 Bundle format defined.** `manifest.json` + `bundle.db` +
      `trust-anchors.toml` schema is documented in
      `data/community/README.md` and pinned by Go types in
      `shell/internal/cache/community/bundle.go`. The format includes
      `version`, `created_at`, `intent_count`, `signer_id`, `signature`
      (base64 over SHA-256 of `bundle.db`), and the `os`-bucketed
      contents of the SQLite file (same columns as the L1 store, no
      stats rows).
- [ ] **#60 Curated seed of common intents.** `data/community/seed.jsonl`
      ships with **at least 100 curated (intent, os, invocation) rows**
      across darwin/linux/windows. The 1000+ stretch is acknowledged
      out of scope for this PR (see Out of Scope) ŌĆö the format is the
      load-bearing artifact; the seed grows in follow-ups.
- [ ] **#61 Bundle ships with the installer.** `make bundle` produces
      `dist/community/aish-community-bundle-vN.tar.gz` + manifest +
      signature. The shell looks for the bundle at three paths in
      order: `$AISH_COMMUNITY_BUNDLE_DIR`, `/usr/local/share/aish/community/`,
      `<binary>/../../share/aish/community/`. First-run install copies
      the verified bundle.db to `~/.aish/community-bundle.db`. Idempotent:
      second run with the same release SHA is a no-op.
- [ ] **#62 Opt-in contribution flow stub.** `aish community contribute
      <intent>` records an opt-in submission to
      `~/.aish/community-contribute.jsonl` with **no network call**.
      The real upload pipeline is out of scope for this PR; the
      built-in lays the contract (privacy: only the intent + invocation
      + os, no cwd / env / args / userid) and the JSONL file format.
      The contribute command is **off by default**; the user must
      explicitly run it per-intent.
- [ ] **#63 Ed25519 signature trust.** Verification uses the stdlib
      `crypto/ed25519` Verify against compiled-in trust anchors
      (`community/trust.go`). A tampered `bundle.db` byte yields
      `community: signature verification failed` on stderr and the
      install path aborts. A signature from an off-anchor key yields
      `community: signer <id> not in trust anchors`. The trust anchor
      list supports a `revoked` flag ŌĆö a revoked signer's bundles fail
      verification regardless of signature validity.
- [ ] **Shell wiring.** On `New()`, the shell calls
      `community.OpenBundle()` after `openCache`. Bundle absence,
      verification failure, and verify-then-load failure are ALL
      best-effort silent (mirroring the existing cache/history/telemetry
      pattern). `Cache.Resolve` consults the community bundle on L1
      miss BEFORE walking the plugin path; an L3 hit promotes to L1
      with `source=community`.
- [ ] **L1 takes priority over L3.** Same intent in both: L1's
      invocation wins. Test-asserted.
- [ ] **Source provenance.** The L1 `intents` table gains a `source`
      column with default `'plugin'`. L3 promotions write `'community'`.
      `aish community info` reports the count of community-sourced
      rows.
- [ ] **`aish community` built-in.**
      - `aish community info` ŌĆö bundle path, version, created_at,
        signer id, intent count, install state.
      - `aish community status` ŌĆö pretty status line (mirrors
        `aish cache stats`).
      - `aish community install [--force]` ŌĆö re-run the install path
        from a fresh bundle on disk.
      - `aish community refresh` ŌĆö alias for `install --force`.
      - `aish community contribute <intent>` ŌĆö opt-in submission.
- [ ] **`make ci` passes** from the worktree.
- [ ] **`make build && make bundle`** produces both the `aish` binary
      and a signed community bundle tarball.

|---|---|---|---|
| **SQLite (chosen)** | Same shape as the L1 store; brute-force `Cosine` similarity already implemented in `store.go`; no new query engine. Atomic file. Cross-platform. | A binary blob is opaque to grep / diff; the build step needs sqlite3 (`modernc.org/sqlite` already vendored, so no new dep). | **Chosen** |

| Custom binary | Smallest on disk; deterministic byte layout makes signing simpler. | New serializer + tests + endianness footguns. No diff path. Forecloses any future "open the bundle in DBeaver" workflow. | Rejected |

## Threat Model

- **Tampered bundle.** Attacker flips a byte in `bundle.db` (e.g., a
  poisoned `rm -rf /` for an innocuous intent). Signature verification
  recomputes SHA-256 over the on-disk bytes and `ed25519.Verify`
  rejects. **Mitigation:** mandatory verify before copy-to-`~/.aish/`;
  no silent fallback. **Test:** `TestVerifyRejectsTamperedBundle` flips
  one byte after signing and asserts the verifier returns the typed
  `ErrSignatureInvalid` error.
- **Off-anchor signer.** Attacker generates their own keypair and signs
  a malicious bundle. **Mitigation:** trust anchors are compiled into
  the binary at build time; an unknown `signer_id` returns
  `ErrUnknownSigner`. **Test:** `TestVerifyRejectsUnknownSigner`.
- **Compromised key.** A legitimate signing key leaks. **Mitigation:**
  trust anchors carry a `revoked` boolean; a revoked signer fails
  verification regardless of signature validity. Anchors are pinned at
  build time so a remote compromise of the anchor list itself requires
  a new aish release ŌĆö which is the right blast radius. **Test:**
  `TestVerifyRejectsRevokedSigner`.
- **Stale bundle.** `manifest.json.created_at` carries a timestamp; the
  loader warns on stderr when the bundle is older than 90 days. The
  90-day warning is informational, not fatal ŌĆö a user without network
  access still needs the L3 cache. **Test:**
  `TestLoadWarnsOnStaleBundle`.
- **Bundle replay (downgrade).** Attacker hands you an old-but-signed
  bundle with a known weakness. **Mitigation:** version number in
  manifest; the loader records the installed bundle's version under
  `~/.aish/community-bundle.json` and refuses to install a strictly
  older version unless `--force` is passed. **Test:**
  `TestInstallRefusesDowngrade`.
- **Path-traversal on bundle dir.** The lookup walks
  `$AISH_COMMUNITY_BUNDLE_DIR` etc. **Mitigation:** the loader treats
  the bundle dir as untrusted: only reads `manifest.json` +
  `bundle.db` + `signature` files by name; rejects any path that
  escapes the directory via symlink (`filepath.EvalSymlinks` +
  containment check). **Test:** `TestLoadRejectsSymlinkEscape`.
- **L3 row poisons L1.** A community row with a destructive
  `invocation` gets promoted to L1 and then executes. **Mitigation:**
  L3 contents pass through the same dispatch pipeline as L1; promotion
  does NOT bypass the v0.1-4 history snapshotter / interceptors. The
  signing key is the trust boundary ŌĆö the upstream curators are the
  ones responsible for not shipping `rm -rf /`. We document the policy
  in `data/community/README.md`. Not a verifier-level mitigation; a
  policy mitigation.

## Approach

1. **Schema migration.** Add the `source` column via append-only DDL +
   `ALTER` for upgrade. Lands as one commit with a unit test asserting
   pre-existing DBs upgrade cleanly.
2. **Verifier subpackage.** Build `community/{bundle,verify,trust}.go`
   in isolation; tests use a freshly-generated test keypair (the dev
   anchor). Lands as one commit; no shell wiring yet.
3. **Loader + Lookup.** Glue verifier to disk: discovery walk, copy
   verified bundle.db to `~/.aish/`, write sidecar `community-bundle.json`,
   `Bundle.Lookup` (read-only SQLite query on bundle.db).
4. **Wire into `cache.Cache.Resolve`.** L1 miss ŌåÆ L3 lookup ŌåÆ on hit,
   promote to L1 with `source='community'`. Tests assert L1-wins-over-L3
   and provenance.
5. **`aish community` built-in.**
6. **Seed bundle.** Hand-curate Ōēź100 (intent, os, invocation) rows
   across darwin/linux/windows in `data/community/seed.jsonl`.
7. **`cmd/aish-community`** + `make bundle` target ŌĆö compiles seed
   into a signed bundle on disk.

## Risk Assessment

- **Crypto correctness.** Mitigated by sticking to stdlib
  `crypto/ed25519` + `crypto/sha256` with no creative wrapping. The
  verifier is < 50 lines; tests cover every error branch.
- **SQLite schema migration.** Adding a column to a live DB on
  upgrade is the historic foot-gun. Mitigated by `ALTER TABLE IF NOT
  EXISTS COLUMN` + a `PRAGMA table_info` guard. Tested via the
  upgrade test.
- **Bundle path discovery on macOS / Linux / Windows.** Mitigated by
  treating discovery as a list of candidate dirs (env override first,
  then well-known paths). All paths are absolute; the loader does
  `filepath.EvalSymlinks` and a containment check before opening any
  file.
- **PR scope.** The 1000+ intent goal is explicitly deferred ŌĆö 100 is
  the MVP floor per the task brief.

## Backward Compatibility

- An existing `~/.aish/cache.db` that lacks the `source` column gets
  the column added by the upgrade migration. Existing rows default to
  `'plugin'`.
- A user who never installs aish v0.2-3 sees no behavior change.
- The new `aish community` built-in is additive.
- `Cache.Resolve` semantics are unchanged when no bundle is loaded ŌĆö
  the L3 tier is skipped entirely.
- The known-binary tier still wins over everything in dispatch
  (`shell.go` line 586).

## Out of Scope

- **1000+ curated intents.** Stretch goal; not in this PR. 100 is the
  MVP floor.
- **Telemetry on L3 hit rate.** Cache stats track L1; an L3-specific
  hit counter is a follow-up. (`aish community info` reports the
  static row count, which is enough.)
- **P2P / federated cache sharing.** Out of scope; trust-anchor
  pinning explicitly precludes it.
- **Real upload pipeline behind `aish community contribute`.** The
  built-in writes to a local JSONL file; the upload daemon is a
  separate task.
- **HTTPS bundle download.** Deferred to v0.2-3.1.
- **Public signing key infrastructure.** Production anchors land via a
  separate PR (cryptographic key management is governance, not code).
