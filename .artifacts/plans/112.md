## Acceptance criteria (full set)

- **AC1 — Embed-on-write.** Every new non-tainted event embeds at write time.
- **AC2 — Vector storage.** Embeddings keyed by `events.id`. Additive schema; pre-#112 DBs open clean.
- **AC3 — Semantic surface.** `aish history search "<query>"` returns top-K by cosine similarity.
- **AC4 — Taint safety.** No tainted body embedded in plaintext (skip, not redacted-embed — see §Risk).
- **AC5 — Hybrid ranking.** RRF (k=60) merges FTS5 and vector hits. `--mode={keyword,semantic,hybrid}`, default `hybrid`.
- **AC6 — Reindex.** `aish history reindex` embeds existing events; idempotent; resumable mid-run.
- **AC7 — Model versioning.** Per-row `model_id` and `dim`; queries reject mixed-model rows; reindex re-embeds on model change.
- **AC8 — Latency.** p50 < 50ms, p95 < 150ms for top-10 over 10K events on a 2023 MBP M2. Bench fails CI on SLO miss.
- **AC9 — Zero network.** No outbound HTTP/HTTPS/DNS during embed or query.
- **AC10 — Migration safety.** Pre-#112 DB opens; FTS5 still works; one-time hint to `aish history reindex` for users with empty vector store.

## Plan addendum (post-Phase-A, 2026-05-21) — T2 backend swap

### sqlite-vec → chromem-go (pure Go)

Phase A discovered `modernc.org/sqlite v1.50.1` (the driver `shell/` uses) **cannot load C extensions**. sqlite-vec and sqlite-vss both require C sqlite + extension loading. The seed handles this gracefully but no real semantic search runs on this driver.

**Decision (2026-05-21, Thomas).** Drop sqlite-vec entirely. T2's `VectorStore` implementation becomes pure-Go — primary candidate **chromem-go** (`github.com/philippgille/chromem-go`). Alternative if chromem-go's API is awkward: `github.com/viterin/vek` SIMD flat-scan + bbolt persistence. The TL has authority to swap if a problem surfaces during implementation.

**Rationale.** Keeps aish's pure-Go portability story. Windows v1.0 ships "free" (no cgo cross-compile pain). The seed interfaces (`EmbeddingProvider`, `VectorStore`, `VectorHit`) are unchanged — only the implementation behind `VectorStore` changes.

**Affected sub-tasks.**
- **T2** — rewrite. Implementation files become `vector_store_chromem.go` (or similar). The `events_vec` virtual-table migration is retired. The `events_vec_meta` sidecar in the seed becomes load-bearing (carries `model_id`, `dim`, last-rebuilt timestamp). Persistence path: `~/.aish/history-vectors/` (chromem-go's gob format), kept separate from `history.db` so two storage formats don't share one file.
- **T1, T3, T4, T5** — unchanged. Continue to depend on the `VectorStore` interface as the seed defined it.

AC1..AC10 still hold — they're interface-level. AC8's latency must be re-confirmed against chromem-go in T2 bench.

## Sub-tasks (parallel-safe; OWNS lists disjoint)

### T1 — Embedding provider (local model integration)
- OWNS: `shell/internal/history/embedding_fastembed.go`, model-load helper, `~/.aish/models/<model-id>/` cache directory layout. Extend `embedding_test.go`.
- Chosen embedder: **fastembed-go + bge-small-en-v1.5** (384-dim, ~30MB ONNX, pure-Go).
- First-run UX (approved 2026-05-21): model is a manual prereq for v0.3 — no `aish history models pull` subcommand. Clear "model not present at <path>" error if missing.
- Acceptance: deterministic embedding, batch order preserved, `Dim()` matches model card, offline-first (no network on cold start).

### T2 — Vector store (chromem-go, pure Go)
- OWNS: `shell/internal/history/vector_store_chromem.go`, `vector_store_persistence.go`, extend `vector_store_test.go`, the chromem-go dep in `shell/go.mod`.
- Persistence: `~/.aish/history-vectors/` (chromem-go gob format).
- Schema linkage: `events_vec_meta` (already in seed) stores `model_id`, `dim`, last-rebuilt timestamp. Vector data itself lives in the chromem-go store, NOT in sqlite.
- If chromem-go's `Collection.Add`/`.Query` API doesn't match `VectorStore.Upsert`/`Query` cleanly, the TL may swap to `viterin/vek` + bbolt — record the swing in the T2 commit message.
- Migration: `events_vec` (sqlite-vec) probe in `schema.go` becomes a confirmed no-op (still helpful for logging "no extension available").
- Acceptance: round-trip upsert→query at rank 1, idempotent re-upsert, delete removes from search, p50/p95 bench passes.

### T3 — Write-path integration (embed-on-Append)
- OWNS: `shell/internal/history/embed_writer.go`, hook into `store.go::Append` between SQL insert and return.
- Best-effort: embedder failure does NOT block history write. Missing vector is recoverable via reindex.
- Tainted events: SKIP entirely (not embed-redacted-placeholder — would cluster all tainted events into one near-perfect neighborhood; side channel).
- Acceptance: embeds non-tainted, skips tainted, no-op when `embedder == nil`, write succeeds even when embed errors.

### T4 — Search surface (semantic + hybrid)
- OWNS: extension to `shell/internal/history/search.go` (new `SemanticSearch` + `HybridSearch` methods; existing `Search` untouched), `shell/internal/shell/builtin_history.go` `--mode` flag wiring, extend `builtin_history_semantic_test.go`.
- Hybrid: RRF k=60 fusing FTS hits and vector hits.
- `--mode=keyword` recovers exactly the pre-#112 path for users who want determinism.
- Empty-vector-store path: friendly "run `aish history reindex`" hint when semantic mode is requested with no vectors yet; hybrid degrades to keyword silently.

### T5 — Reindex command + backfill
- OWNS: `shell/internal/history/reindex.go`, `reindex` subcommand block in `builtin_history.go`, extend `reindex_test.go`.
- Reindex UX (approved 2026-05-21): print a one-time hint, do NOT auto-backfill on Open(). Auto-run would spike CPU on first launch for users with large histories.
- Resumable mid-run via per-row transaction.
- Model-mismatch path: re-embed old rows, delete in same transaction.

## Alternatives Table

### Vector storage (post-decision)

| Alternative | Verdict |
|---|---|
| sqlite-vec extension | **Rejected** — incompatible with `modernc.org/sqlite` (no extension loading). |
| sqlite-vss fallback | **Rejected** — same incompatibility. |
| Pure-Go flat scan (no index) | Rejected — O(n) scan dies past ~50K events. |
| **chromem-go (pure Go embedded vector DB)** | **Chosen.** Persists to disk; cosine; permissive license; no cgo. |
| `viterin/vek` SIMD + bbolt | Fallback if chromem-go's surface bites against `VectorStore` interface. |
| Hosted (OpenAI / Voyage / Cohere) | Rejected for v0.3 (local-only mandate); #203 follow-up. |

### Embedding generator (unchanged)

| Alternative | Verdict |
|---|---|
| **fastembed-go + bge-small-en-v1.5** | **Chosen.** Pure-Go ONNX; 384-dim; deterministic. |
| llama.cpp embedding mode | Rejected v0.3 (couples to #93). |
| Shell out to `ollama embed` | Rejected (too many if-installed branches). |
| sentence-transformers (Python subprocess) | Rejected (Python runtime as dep — unacceptable). |

## Test strategy

- Unit: vector-store CRUD; embedder determinism; taint-aware skip; RRF math on synthetic rank lists.
- Integration: full ingest → query loop against a real chromem-go store (no mocks).
- Property: similarity ranking stable under shuffled insertion; reindex on already-reindexed DB is a fixed point.
- Benchmark: `BenchmarkVectorStore_Query10K` enforces AC8 (p50 < 50ms, p95 < 150ms).
- Migration: fixture pre-#112 `history.db` under `shell/internal/history/testdata/` (already in seed); Open succeeds, FTS5 still works.
- Adversarial focus:
  - Taint leakage into vector blobs.
  - Migration idempotency (Open ten times → schema delta zero after first).
  - Reindex interruption (kill mid-run; restart → completes).
  - Model drift (no mixed-model query results).
  - Embedding-of-redacted-form would be a side channel — assert SKIP behavior holds.

## Risks

- chromem-go's API maturity (May 2026) — surface confirmed in §Alternatives, fallback documented.
- Windows v1.0 cross-compile — pure-Go path eliminates the cgo concern entirely (this was a driver for option C).
- Storage growth: 384 × 4B × 10K = 15MB (fine). 100K = 150MB (v0.4 review).
- Best-effort embedding (failure-non-blocking) keeps semantic search a value-add, not a critical-path dep.

## Backward compatibility

Additive schema only. Existing FTS5 path untouched. `--mode=keyword` recovers pre-#112 behavior. Reindex is opt-in (one-time hint on first semantic search if vectors absent).
