# Plan: v1.0-2 (Win32 OS Translation) + v1.0-3 (Windows Script Translation)

> Two adjacent epics in the `domain:os-translation` partition. Together
> they give Windows users (a) Windows-native built-ins for the most
> common admin verbs (install / service / process / env / network) and
> (b) the ability to feed PowerShell or `cmd`/`bat` scripts into the
> existing `aish run | explain | migrate` pipeline. MVP-first; depth
> deferred aggressively.

**Source:** GOALS.md §"Epic v1.0-2 — Win32 OS Translation",
§"Epic v1.0-3 — Windows Script Translation".
**Epics (supervised):** #135, #142.
**Tasks (in-scope this PR):** #136, #137, #138, #139, #140, #141, #143, #144, #145.
**Domain:** `domain:os-translation`.
**Risk:** HIGH (v1.0-2 — Win32 surface area); MEDIUM (v1.0-3 — net-new
readers but the AST + engine are stable).
**Milestone:** v1.0 — Windows Native (120 days).
**Sibling TLs running in parallel:** TL_WIN_BUILD (#130-#134, Makefile +
CI), TL_WIN_SECRETS (#146-#147, credential manager), TL_WIN_LOGIN
(#148-#150, ConPTY + console host). Boundaries enforced via the per-TL
file-ownership lists in the master plan.

## Acceptance Criteria

### v1.0-2 — Win32 OS Translation

- [ ] **#136 Win32 bindings (subset needed).** New file
      `shell/internal/exec/win32.go` (build tag `//go:build windows`)
      that wraps the subset of `golang.org/x/sys/windows` APIs every
      v1.0-2 built-in needs: process enumeration, service control,
      environment block reads, network adapter info. A matching
      `win32_other.go` (build tag `!windows`) returns
      `exec.ErrUnsupported` for every entry point so the rest of the
      package compiles on macOS / Linux.
- [ ] **#137 `aish install <pkg>`.** New built-in delegating to the
      `winget` CLI. On non-Windows hosts, prints "install: not
      supported on <GOOS>" and returns 2.
- [ ] **#138 `aish service <list|status|start|stop> [name]`.** New
      built-in that calls the SCM (`OpenSCManager`, `EnumServicesStatusEx`,
      `OpenService`, `QueryServiceStatus`, `StartService`,
      `ControlService`) via `x/sys/windows`. MVP scope: read-only
      `list` + `status`; mutations (`start`/`stop`) surface a
      one-line warning + require elevated SCM access.
- [ ] **#139 `aish process <list|kill> [pid]`.** New built-in.
      `list` walks the snapshot via `CreateToolhelp32Snapshot` +
      `Process32First/Next`. `kill <pid>` calls `OpenProcess` +
      `TerminateProcess`. MVP: PID + ProcessName + parent PID; CPU/
      memory rolled into a follow-up.
- [ ] **#140 `aish env <list|get|set|unset> [name [val]]`.** New
      built-in. `list`/`get` read the process environment block (no
      Win32 needed — `os.Environ` is the same data). `set`/`unset`
      mutate the in-process env via `os.Setenv`/`os.Unsetenv` AND, when
      `--persist` is supplied, write through to the user's environment
      block via `SetEnvironmentVariable` + a `WM_SETTINGCHANGE`
      broadcast via `SendMessageTimeout`. MVP: in-process only;
      `--persist` is a v1.1 follow-up explicitly deferred.
- [ ] **#141 `aish network <interfaces|routes>`.** New built-in. MVP:
      `interfaces` lists adapter name + MAC + first IPv4 + status via
      `GetAdaptersAddresses`. `routes` is deferred to v1.1 with a
      clear "not yet implemented" message.
- [ ] **Cross-compilation gate.** `GOOS=windows GOARCH=amd64
      GOFLAGS=-buildvcs=false go build ./shell/...` succeeds; native
      macOS `make ci` stays green.

### v1.0-3 — Windows Script Translation

- [ ] **#143 PowerShell reader.** New file
      `shell/internal/translate/reader/powershell.go`. Token-boundary
      lexer (no full PS grammar — mvdan.cc/sh has no PS support and
      vendoring tree-sitter is out of scope). MVP surface:
      `Write-Host`/`Write-Output`, `Set-Variable`/`$x = val`, simple
      pipelines (`cmd1 | cmd2`), `if/else`, comments (`#` and
      `<# … #>` block), explicit `function name { … }`. Anything else
      → `Unknown` with the original source preserved.
- [ ] **#144 cmd/bat reader.** New file
      `shell/internal/translate/reader/cmd.go`. MVP surface:
      `echo …`, `set NAME=VAL`, `if [not] … ( … ) [else ( … )]`,
      label-less single-statement lines, comments (`REM` and `::`),
      `goto :EOF` recognised as a no-op exit. Pipelines via `|` map
      to translate.Pipe. Everything else → `Unknown`.
- [ ] **#145 Test against common Windows admin scripts.** Golden
      `win_testdata/` directory with representative samples (winget
      install loop, service restart, env-set, simple `if errorlevel`
      cmd script) — each parses, `aish explain` produces a numbered
      description, `aish migrate` emits aish-native text without
      hanging.
- [ ] **Reader dispatch wiring.** `translate.Read` learns
      `DialectPowerShell` and `DialectCmd`; `Detect` picks them up via
      shebang (`#!/usr/bin/env pwsh`), extension (`.ps1`, `.psm1`,
      `.bat`, `.cmd`), and a lightweight content heuristic.
- [ ] **`make ci` passes** with `GOFLAGS=-buildvcs=false`.

## Rationale & Alternatives

### Win32 bindings: `x/sys/windows` direct vs CGO

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **`golang.org/x/sys/windows`** (chosen) | Pure-Go; already an indirect dep; cross-compiles from macOS; minimal new surface. | Coverage is broad but not 100% of Win32; thin wrappers needed. | **Chosen** — preserves the no-CGO promise the project has held since v0.1-1. |
| Vendored CGO bindings (`go-ole`, `win32` C-headers) | Direct access to every Win32 API. | Breaks no-CGO promise; complicates the build matrix; needs a Windows toolchain to cross-compile from macOS. | Rejected. |
| Shell out to PowerShell for everything | Trivial to ship; matches what users already know. | Defeats the purpose — aish is supposed to be the shell, not a PowerShell front-end. Adds 200-500 ms startup latency per invocation. | Rejected. |

### Built-in shape: per-subcommand vs single dispatcher

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **Per-subcommand built-ins** (`aish install`, `aish service`, …) (chosen) | Matches the verb-shaped catalog in GOALS.md; each built-in stays in one file (~50-150 lines); easy to map to issues. | 5 new files in `shell/internal/shell/`. | **Chosen** — file-per-verb mirrors the rest of the codebase. |
| Single `aish win <verb>` dispatcher | One entry point; less surface in shell.go. | Hides the verbs behind a Windows-shaped namespace, contradicting GOALS' "OS insensitive" framing — `aish install` should work the same on every platform (even when the implementation underneath is winget vs apt vs brew). | Rejected. |
| Reuse OS-native binaries via `runExternal` | Zero new code. | No structured output; no portability layer; reduces aish to a passthrough. | Rejected. |

### PowerShell reader: lex on token boundaries vs full grammar

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **Token-boundary lexer + heuristic statement parse** (chosen) | Ships in days, not weeks; matches the MVP discipline of the bash reader; unknown constructs surface as `Unknown` rather than crash. | Will miss expressions inside `$(...)`, advanced parameter binding, `param()` blocks, etc. | **Chosen** — explicitly an MVP, with the deferral list documented. |
| Vendor tree-sitter PowerShell grammar | Full coverage; correct semantics. | Adds a CGO/wasm dependency, ~5-15 MB of grammar artifacts, ~1 week of integration work. | Rejected for v1.0; revisit at v1.1 if user demand warrants. |
| LLM-only intent extraction | Zero parser code. | Per-script API cost; non-deterministic; contradicts the `aish migrate` reproducibility requirement. | Rejected. |

### Cmd/bat reader: handwritten vs reuse PowerShell lexer

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **Handwritten line-oriented parser** (chosen) | Cmd is line-oriented and tiny — `echo`/`set`/`if`/`goto`/`REM` covers >90% of "ordinary" .bat files. ~100-150 LoC. | Net new code. | **Chosen**. |
| Treat cmd as a dialect of PowerShell | Single reader to maintain. | Syntax is materially different (`%VAR%` vs `$VAR`, `REM` vs `#`, `IF NOT` vs `if (-not …)`). Would force mode flags. | Rejected. |

## Scope Contract (MVP)

**In scope this PR.**

Built-ins (all Windows-only at runtime; non-Windows hosts get
`exec.ErrUnsupported` or a one-line "not supported" message):

- `aish install <pkg>` — `winget install <pkg>` (passthrough).
- `aish service list` — enumerate services via SCM.
- `aish service status <name>` — `QueryServiceStatus`.
- `aish service start|stop <name>` — `StartService` / `ControlService`.
- `aish process list` — toolhelp snapshot.
- `aish process kill <pid>` — `OpenProcess` + `TerminateProcess`.
- `aish env list|get|set|unset` — in-process only.
- `aish network interfaces` — adapter addresses.

Readers:

- PowerShell: `Write-Host`/`Write-Output`, variable assignment,
  pipelines, single-arm `if/else`, comments, function definitions.
- Cmd: `echo`, `set`, `if/else`, comments, `goto :EOF`, pipelines.

**Out of scope this PR (explicit deferrals).**

- `aish env --persist` (HKCU + `WM_SETTINGCHANGE` broadcast) → v1.1.
- `aish network routes` → v1.1.
- `aish process` CPU / memory columns → v1.1.
- `aish service` log streaming / `query` filter expressions → v1.1.
- PowerShell `param()` blocks, splatting, advanced parameter binding,
  `try/catch` → v1.1.
- Cmd `for /F`, `call`, labels other than `:EOF`, `setlocal` →
  surfaced as `Unknown`.
- Service mutation tests (require elevated runtime) → manual smoke on
  a Windows VM by the user, not gated in CI.

## Architectural Shape

```
shell/internal/exec/
├── win32.go               //go:build windows   — x/sys/windows wrappers.
├── win32_other.go         //go:build !windows  — ErrUnsupported stubs.
└── (existing files untouched)

shell/internal/shell/
├── builtin_install.go     dispatch + winget shell-out.
├── builtin_service.go     SCM glue.
├── builtin_process.go     toolhelp snapshot + kill.
├── builtin_env.go         env table.
├── builtin_network.go     adapter listing.
├── builtin_win_test.go    in-process tests with the exec stubs.
└── shell.go               additive: 5 new dispatch branches.

shell/internal/translate/
├── ast.go                 +DialectPowerShell, +DialectCmd consts.
├── translate.go           Readers struct +PowerShell / +Cmd fields;
│                          Read/Detect learn the new dialects.
└── reader/
    ├── powershell.go      lexer + statement-shape parser.
    ├── powershell_test.go
    ├── cmd.go             line-oriented parser.
    ├── cmd_test.go
    └── win_testdata/      sample .ps1 / .bat scripts.
```

The win_testdata directory lives under `reader/` (not the existing
shared `testdata/`) so the bash/zsh/fish tests don't accidentally
pick up Windows fixtures.

## Testing strategy

- **Win32 wrappers (#136).** Unit-test the non-Windows stubs from
  macOS (`go test ./shell/internal/exec/...` on darwin returns
  ErrUnsupported deterministically). The Windows-tagged path is
  compile-gated via `GOOS=windows go build ./shell/...` — runtime
  verification is delegated to a Windows VM smoke test post-merge.
- **Built-ins (#137-#141).** Inject a function-typed seam for the
  Win32 calls so the built-in's *dispatch* logic (argv parsing,
  error formatting) is unit-testable on macOS without a Windows
  runtime. The seam mirrors the `execFn` pattern already in
  shell.go for `exec`.
- **Readers (#143, #144).** Golden-file parse tests under
  `win_testdata/`. Each fixture has expected AST shape assertions
  in the table-driven test. `Unknown` count and reasons asserted.
- **End-to-end smoke (#145).** A test that runs all fixtures
  through `translate.Detect → Read → Explain → Migrate` and
  asserts no panic, deterministic output, and a non-empty migrate
  result for each.

## Risk Assessment

- **Win32 API surface drift.** `x/sys/windows` covers most of what
  we need but a missing symbol forces a CGO escape hatch. Mitigation:
  scope is deliberately small; if a symbol is missing we use
  `syscall.NewLazyDLL` + `NewProc` (pure-Go syscall trampoline) as a
  fallback before reaching for CGO.
- **Cross-compilation surprises.** macOS dev box cannot run the
  Windows code. Mitigation: `GOOS=windows go build` gate is required
  before commit; runtime smoke is post-merge.
- **PowerShell lexer false-positives.** A token-boundary lexer can
  misread quoted strings containing `|` or `=`. Mitigation: extensive
  fixture-based regression tests; everything ambiguous becomes
  `Unknown` rather than half-parsed.
- **Elevation / privilege.** Service mutations require elevated
  process. Mitigation: surface the privilege error verbatim — do not
  attempt UAC prompting from the shell process.

## Dependencies

- `golang.org/x/sys/windows` — already an indirect dep; promote to
  direct in `shell/go.mod`. No new module add.
- Sibling TL boundaries:
  - TL_WIN_BUILD owns the Makefile and CI cross-compile gate — this
    PR only adds source files that the gate covers.
  - TL_WIN_SECRETS owns the credential manager wrapper — no overlap.
  - TL_WIN_LOGIN owns ConPTY — `pty_windows.go` is theirs to extend;
    we leave it untouched.

## Backward compatibility

- All new built-ins are additive — existing dispatch branches are
  untouched. The new `case` arms in `shell.go` insert between the
  existing built-ins and the AI-dispatch fallback.
- Reader dispatch is additive: the `Readers` struct grows two
  fields, `translate.Read` learns two new cases. Tests of the
  existing readers are unchanged.
- `DialectPowerShell` and `DialectCmd` are new constants — no
  existing const renumbered.
- Non-Windows hosts: every new built-in returns a clear "not
  supported on <GOOS>" message and exit 2, with no panic, no
  crash, and no startup-time impact.
