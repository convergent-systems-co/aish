## Acceptance Criteria

- [ ] **#39** Opt-in gate at `~/.aish/telemetry.toml`. Two independent
      switches:
      `opt_in_local = true` (default) — own machine, own data, always on.
      `opt_in_aggregate = false` (default) — outbound to team dashboard,
      off until user explicitly enables. First-run prompt is **out of
      scope** for v0.1-5; the default opt-out posture IS the consent
      gate. The file is auto-created with documented defaults on first
      successful `Recorder.New`, with a header comment explaining each
      key.
- [ ] **#40** Per-session counters: `Commands`, `CacheHits`, `CacheMisses`,
      `InferenceCalls`, `ExternalRuns`, `FailedCommands`, `WallTime`.
      Computed from the cache.Store stats *delta* observed at session
      open vs each After interceptor call (no mutation of cache/),
      combined with the cost-log JSONL since session start.
- [ ] **#41** Cache hit-rate time series: on `Recorder.Close`, append a
      session row to `~/.aish/sessions/<id>.json`. `aish stats` renders
      the last N sessions (default 10, configurable via
      `telemetry.history_window`).
- [ ] **#42** Inference cost tracking in USD. Reads
      `~/.aish/cost-log.jsonl` (plugin owns the write side; we never
      append). Filters rows by chronon `>=` session start to avoid
      double-counting earlier-session entries. Per-model breakdown is
      computed but only the session total renders in the default `stats`
      table. Drachma: GOALS.md mentions Drachma alongside USD but the
      Convergent Systems Drachma exchange rate is not pinned in v0.1.
      USD only for v0.1-5; a `TODO(v0.3-economy)` comment names the gap.
- [ ] **#43** `aish stats` built-in renders a stable, parseable table:
      one row per session for the last N sessions plus a footer with
      cumulative cache hit-rate and total USD across the window. Empty
      window renders the header + a single `(no sessions recorded yet)`
      line, never an error.
- [ ] **#44** Aggregate dashboard transport — when `opt_in_aggregate`
      is true, on `Recorder.Close` write the session payload to
      `~/.aish/sessions/pending/<id>.json`. The actual HTTP POST to
      `api.convergent-systems.co/telemetry/v1/sessions` is deferred
      (the endpoint isn't live yet). When the flag is false, no
      pending file is written. Local session files (#41) still write
      regardless — they're own-machine data.
- [ ] Telemetry interceptor registers on `s.interceptors` exactly once
      (after history per TL1's seam). `Before` is a no-op (`return
      nil`); `After` records exit + duration + delta cache stats.
- [ ] `make ci` passes at repo root.
- [ ] Live smoke (REPL) prints non-error session row with `cmds >= 2`
      after two `echo` invocations.

## Telemetry Interceptor (After-only)

```go
// telemetry.Recorder implements shell.Interceptor.
func (r *Recorder) Before(_ *parser.Pipeline, _ string) error {
    return nil  // record-only; no veto on commands
}

func (r *Recorder) After(pl *parser.Pipeline, line string, exit int, dur time.Duration) {
    r.session.Commands++
    r.session.WallTime += dur
    if exit != 0 {
        r.session.FailedCommands++
    }
    // Delta from previous After's cached snapshot of cache.Store().Stats().
    // Two int64 reads from a SQLite-backed Stats() call — cheap.
    // Falls through silently if cacheStore is nil (shell ran without cache).
    r.observeCacheDelta()
}
```

The cache-delta approach keeps us out of `shell/internal/cache/` and out
of the Interceptor contract itself, exactly per the boundary TL1
documented in `interceptor.go`.

## Alternatives Table — Opt-in storage

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|
| **A. `~/.aish/telemetry.toml` file with two booleans + auto-create defaults** | Single source of truth; user-editable; mirrors the existing `~/.aish/config.toml` pattern from v0.1-4; survives reinstalls. Default opt-out is enforceable simply by absence. | Requires a tiny TOML reader/writer (no nested types — just two key=bool lines, so ~50 LOC). | **CHOSEN** |
| **B. Env var only (`AISH_TELEMETRY=on`)** | Zero file I/O. Trivial. | No persistence across shell sessions unless user adds to their `.zshrc`; loses the user's choice on a fresh shell. Bad UX for the privacy story. | Rejected. |
| **C. First-run interactive prompt** | Visible consent moment. | Requires interactive stdin; breaks scripted use (`HOME=$(mktemp -d) ./aish <<EOF`); first-run detection is brittle (what if `~/.aish` already exists from a prior version?). | Rejected for v0.1-5; the file's documented defaults stand in for the prompt. Revisit in v0.2 with the TUI work. |

Choice: **A**. The two-boolean file is the smallest surface that
preserves user agency across sessions, supports scripted use, and lets
the user audit their consent state by reading one file.

| Alternative | Pros | Cons | Verdict |
|---|---|---|---|

## Test Strategy

- **Unit per file**, table-driven where helpful.
- **`session_test.go`** — counter math; session ID format
  (RFC4122 v4); zero state.
- **`recorder_test.go`** — full Before/After cycle with an in-memory
  cache.Store stub or a real one against a tempdir SQLite; assert
  cache hit/miss delta attribution; assert After tolerates a missing
  Before (per Interceptor contract).
- **`cost_test.go`** — fixture cost-log with three rows at varying
  chronons; assert filter-by-since picks the right subset; assert
  missing file returns zero; assert malformed JSON lines are SKIPPED
  (not erroring) per "robust readers" discipline.
- **`consent_test.go`** — first-call writes defaults; second call
  returns the existing values unchanged; partial file (one key
  missing) gets the missing key defaulted but the present key honored.
- **`store_test.go`** — round-trip a SessionRow; `ListSessions`
  ordering (most recent first); pending file written iff aggregate
  opt-in; pending NOT written when only local opt-in.
- **`builtin_stats_test.go`** — render against empty store; render
  against one-session store; render against ten-session store; assert
  column alignment is stable (no tabs vs. spaces drift).
- **Adversarial focus areas** (Tester wave):
  - Consent default MUST be opt-out for aggregate. A missing or
    malformed consent file MUST NOT enable aggregate.
  - Cost-log replay: appending a duplicate row to the JSONL must NOT
    double-count *within a single session* (the chronon filter handles
    cross-session; within-session duplicate is the plugin's bug, not
    ours, but the test names it).
  - `aish stats` with no `~/.aish` directory (fresh user) must not
    panic — must render the empty-state line.
  - Telemetry recorder behavior when the cache failed to open
    (`s.cacheStore == nil`): cache delta is zero, but command count
    still ticks. No nil deref.
  - Pending payload contains NO secrets — explicitly assert it has no
    `api_key`, `token`, `path`, or `command_line` fields. Session
    payload schema is documented + tested.

## Risks

| Risk | Mitigation |
|---|---|
| Cache `Stats()` reads on every command add SQLite query overhead | Two SELECTs per command against an indexed `stats` k/v table — sub-microsecond on local SQLite. If profiling later shows a problem, cache the delta in-memory and reconcile less often. Acceptance is correctness, not perf. |
| Consent file race (two shells starting in parallel both write defaults) | Use `os.OpenFile(O_CREATE|O_EXCL)` — second writer fails harmlessly and falls back to read. |
| Cost-log chronon filter mis-parses RFC3339Nano | Plugin emits via `time.RFC3339Nano`; we parse with the same constant. Test asserts round-trip. |
| Pending file dir balloons over time | Out of scope for v0.1-5 (matches the deferred-GC pattern from v0.1-4 snapshots). v0.2 transport work will drain on successful POST. |
| Session ID collision | crypto/rand 128-bit — collision prob is negligible. Filename includes session ID so a collision would be ENOENT-detectable. |
| Drachma fabrication | USD-only per the alternatives table; no Drachma field emitted. |
