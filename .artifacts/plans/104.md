## Acceptance criteria (the four-adapter atomic swap)

- [ ] `aish persona use <name>` atomically swaps {SSH key in agent, cloud profile, kube context, git config user.{name,email,signingkey}} or rolls back all of them when any step fails.
- [ ] Each adapter exercised against fake external state under a sandbox HOME (real adapter code, fake external state — no mocks substituted for the adapter interface).
- [ ] Partial-failure: if adapter N's Apply fails, Rollback runs in reverse order on adapters 1..N-1. Multi-error surfaces both the original cause and any rollback errors.
- [ ] Signed `persona.use` history event records the atomic operation: per-adapter Outcome, prior-state digest. NO secret material in the event body (verified against taint redaction).
- [ ] `aish persona show <name>` reflects declared external bindings.
- [ ] Backward compatibility: a persona with NO `[external]` block behaves identically to pre-#104 (asserted by `TestAtomicPersonaSwitch_NoBindingsPreservesLegacyBehaviour`).

## Sub-tasks (parallel-safe; OWNS lists disjoint)

### T1 — SSH agent adapter
- OWNS: `shell/internal/persona/adapter/ssh.go`, `ssh_fakeagent_test.go`, extend `ssh_test.go`.
- Strategy: `golang.org/x/crypto/ssh/agent` (native Go).
- Schema: `[external.ssh]` → `key_label` (vault label), `add_lifetime_seconds`.
- Capture: `agent.Client.List()` → public-key bytes only. **NO private key in snapshot.**
- Apply: load persona's private key from vault by label; `agent.Client.Add()`.
- Rollback: remove any key whose fingerprint is NOT in the captured set.
- Constraint: zero the private-key buffer via `secrets.Zero(buf)` after Add.

### T2 — Cloud profile adapter (gcloud/aws/azure)
- OWNS: `shell/internal/persona/adapter/cloud.go`, `cloud_gcloud.go`, `cloud_aws.go`, `cloud_azure.go`, extend `cloud_test.go`, `testdata/cloud/*`.
- Strategy: native file-edit floor + Azure shell-out exception.
- Schema: `[external.cloud]` → `gcloud_config`, `aws_profile`, `azure_subscription`.
- **gcloud:** file-edit `~/.config/gcloud/active_config` (plain text single line).
- **aws (approved 2026-05-21):** emit `AWS_PROFILE` env var for the session, NOT file rewrite. Persona's AWS binding is process-scoped, not file-scoped — matches kube's session-wide semantics.
- **azure (approved 2026-05-21):** shell-out to `az account set --subscription`. The file (`~/.azure/azureProfile.json`) has undocumented schema that drifts between az versions; not worth replicating.
- Capture: tar of relevant files + env-var snapshot.
- Rollback: extract tar; restore env vars.

### T3 — Kube context adapter
- OWNS: `shell/internal/persona/adapter/kube.go`, extend `kube_test.go`, `testdata/kube/kubeconfig.yaml`.
- Strategy: `k8s.io/client-go/tools/clientcmd` (native, read+write only, no API server touched).
- Schema: `[external.kube]` → `context`.
- Capture: current-context name + sha256 of merged config (TOCTOU defence).
- Apply: `clientcmd.ModifyConfig`.
- Rollback: write back captured context; if sha mismatch, surface non-fatal warning in Outcome and still restore.
- **Fallback (approved 2026-05-21):** if client-go transitive size pushes the binary beyond ~50MB on darwin, fall back to direct YAML edit via `gopkg.in/yaml.v3` and document the swing in the T3 commit.

### T4 — Git config adapter
- OWNS: `shell/internal/persona/adapter/git.go`, extend `git_test.go`, `testdata/git/gitconfig`.
- Strategy: `exec.Command("git","config","--global", …)` (shell-out — chose A here; git owns its own config file format with includes/conditional-includes that we should not reimplement).
- Schema: `[external.git]` → `scope = "global"` only for v0.3-3, `user_name`, `user_email`, `signing_key`.
- Capture: `git config --global --get user.name` etc.; snapshot is `{name, email, signingkey}` tuple (empty string = "was unset").
- Rollback: re-set captured values; use `--unset` for captured-empty.
- Schema validation: `scope = "local"` is rejected (deferred to v0.3-fu).

### T5 — Orchestrator wiring + persona-builtin integration
- OWNS: `shell/internal/shell/builtin_persona_atomic.go`, extend `builtin_persona_atomic_test.go`.
- One MODIFICATION: `shell/internal/shell/builtin_persona.go::personaSet` (line 269 area) — two-line diff to route through `personaSetAtomic`.
- Approach: new `(s *Shell) personaSetAtomic(name, stdout, stderr) int` that builds the adapter slice from the persona's `ExternalBindings`, runs `Transaction.Execute`, and on success calls `WriteActivePersona` + `recordPersonaUse` with the Outcome attached.
- Mutex (approved 2026-05-21): process-local mutex + sentinel file under `~/.aish/` (stat'd before Apply; warn if recent sentinel exists; don't block cross-process).
- Event body (approved 2026-05-21): embed full Outcome inline in the signed `persona.use` event body (~2KB acceptable). No sidecar.
- **T5 lands LAST** — depends on T1..T4 because its integration test wires all four real adapters.

## Test strategy

- **Unit (each adapter):** Capture→Apply→Verify→Rollback round-trip against fixture HOME/socket. Snapshot has no secret-bearing bytes (adversarial assertion).
- **Unit (orchestrator):** state-machine tests with synthetic adapters; success path, Apply-N fails, Verify-N fails, Rollback-K fails (multi-error surfacing), zero-adapter case (legacy preservation).
- **Integration (T5):** all four real adapters wired against fake external state under one sandbox HOME. NOT mocks. Per Thomas's repeated policy.
- **Adversarial focus (testers):**
  - Partial-failure rollback ordering — fail at every position N in [1..4]; assert reverse-order rollback covers 1..N-1.
  - Race between two `persona use` calls — process-local mutex.
  - History-event leakage — assert SSH private key bytes never appear in event body.
  - TOCTOU on Kube (sha256 of merged config check fires).

## Risks

- **SSH agent state is process-global** — capture-then-restore preserves any user-added keys (rollback enumerates by fingerprint, not "everything except ours").
- **Cloud CLI file-level locking** — `~/.config/gcloud/`, `~/.aws/`, `~/.azure/` are not file-locked; concurrent writes by the actual CLI during a switch interleave. Mitigation: snapshot in memory, last-writer-wins on rollback is acceptable.
- **Kube context affects every kubectl in the session** — by design; documented in `persona show`.
- **Git scope locked to `--global` only** for v0.3-3.
- **Signed event body grows ~200B → ~2KB** with Outcome embedded; acceptable per approval.
- **TOCTOU not in threat model** — "atomic against partial failure," not "atomic against active local adversary." Documented in `adapter/doc.go`.
